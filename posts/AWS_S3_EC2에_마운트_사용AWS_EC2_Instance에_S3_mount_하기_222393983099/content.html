[ Amazon Simple Storage Service ]

EC2에 S3 스토리지 마운트

AWS(Amazon) EC2 Instance에 S3 mount 하기

​

AWS S3를 EC2에 EBS처럼 사용해보자.

​

아마존에서 S3는 매우 유용한 서비스다. 하지만 s3는 object 기반이고 REST http 기반이다 보니 사용함에 있어 많은 불편함이 있다. 그래서 AWS는 최근에 Storage Gateway라는 서비스를 제공한다!

​

하지만 Storage Gateway를 설정하여 사용하려면, Local IDC 환경 또는 AWS에 Gateway appliance 를 VM형태로 올려야 하기 때문에 추가 비용이 들고 간단히 사용하기엔 번거롭다.

​

이때 간단한 해결책으로 S3를 filesystem 처럼 mount 하여 사용하는 방법이 있다.

바로 s3fs. 오픈소스이며 AWS의 공식지원패키지는 아니다.

​

​

1. 관련 프로그램 설치

apt-get update apt install s3fs awscli -y
s3fs

아래와 같이 뜬다면 정상적으로 설치된 것이다.

s3fs: missing BUCKET argument. Usage: s3fs BUCKET:[PATH] MOUNTPOINT [OPTION]...

​

2. 인증 작업(IAM)

Identity and Access Management(IAM) :

오픈스택의 키스톤 같은 보안 증명 서비스다.

​

IAM 키 파일 다운로드

[새 액세스 키 만들기]

​

키 파일 다운로드 클릭해서 받고 열어본다.

파일 내용으로 다음을 작성해 준다.

​

​

3. .s3fs-creds 파일 작성

cd ~ vi .s3fs-creds

​

​

- 파일 권한 변경

chmod 600 ~/.s3fs-creds

​

s3fs [버킷 이름] [특정 디렉토리] [ex] s3fs minji-kit /s3 -o passwd_file=/root/.s3fs-creds

​

- 확인

df -h

​

확인 후 해당 디렉토리로 가보면, 우리가 버킷에 올렸던 파일들이 보인다.

​

여기서 파일을 만들어보고, S3 객체에도 추가되는지 확인한다.

​

​

확인 완료 !
