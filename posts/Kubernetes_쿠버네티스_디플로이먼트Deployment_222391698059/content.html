디플로이먼트(Deployment)?

로드밸런서와 로드밸런서에 연결된 서버들처럼 시스템의 처리 능력을 분산시키고 장애가 발생한 서버들을 제외한 나머지 서버에게 부하를 분산하는 기능을 하는 오브젝트.

파드의 수를 유지시키기도하며 애플리케이션의 버전을 업그레이드할 때 새로운 버전의 파드로 조금씩 바꾸며 업그레이드를 진행할 수 있기 때문에 무중단 없그레이드 가능하고,

보통 디플로이먼트가 생성되면 해당 디플로이먼트에 대응하는 레플리카셋도 함께 생성된다.

따라서, 디플로이먼트를 사용하면 파드와 레플리카셋을 직접 생성할 필요가 없다.

​

​

1) 디플로이먼트의 생성과 삭제

생성 명령어:

kubectl apply -f [yaml 파일]

​

삭제 명령어 :

kubectl delete -f [yaml 파일]

​

Deployment 생성 yaml 템플릿:

apiVersion : apps/v1 설정 kind : Deployment metadata : name에 오브젝트의 이름을 설정 spec : 디플로이먼트의 사용을 기술 replicas : 파드 템플릿을 사용해서 가동할 파드의 수를 지정 selector : matchLabels의 라벨을 통해 실행할 템플릿을 찾는다. template.metadata.labels의 값과 일치하지 않으면 에러 발생 template : metadata: labels: selector에서 지정한 matchLabels의 값과 같은 값을 입력 spec: containers: 컨테이너 스펙을 작성

​

위를 참고하여 nginx-deployment를 생성해 실습을 진행한다.

vi nginx-deployment.yml

​

내용:

apiVersion: apps/v1 kind: Deployment metadata: name: web-deploy spec: replicas: 3 selector: matchLabels: app: web template: metadata: labels: app: web spec: containers: - name: nginx image: nginx:1.16

​

nginx-deployment 생성.

kubectl apply -f nginx-deployment.yml

​

생성 확인:

kubectl get deployment

​

디플로이먼트 자동 복구

worker1 의 전원을 꺼본다.

​

잠시 후 확인해보면 worker1 대신 다른 곳에 pod가 생성된 것을 확인할 수 있다.

​

​

2) 스케일 기능

수동으로 레플리카의 값을 변경하는 방법에 대해 알아보자.

같은 이름의 yaml로 apply하면 새로운 오브젝트가 생성되는 것이 아닌 변경사항이 적용된다.

​

(물론 아래 명령어로도 스케일 변경이 가능하다)

kubectl scale --replicas=[숫자] [디플로이먼트 이름]

​

yml 파일로 스케일을 변경해보자.

vi nginx-deployment.yml

기존 파일에서 수정할 부분인 'replicas: 10' 만 수정 후 저장하고 나간다.

​

​

아래는 deployment 생성 명령어지만, 같은 이름의 파일이 있다면

더 생성되지 않고 수정되는 것을 확인해 보자.

kubectl apply -f nginx-deployment.yml

​

확인:

kubectl get deployments

​

deployment 가 하나 더 생성되지 않고

기존에 있던 web-deploy deployment 내용이 변경되었다.

​

​

3) 롤아웃 기능

kubectl rollout

'롤아웃 기능'은 쿠버네티스 클러스터에 새로운 소프트웨어나 서비스를 개시하는 기능이다.

즉, 컨테이너의 버전을 업데이트하는 것.

​

이번엔 'image: nginx:1.16' 부분을 'image: nginx:1.17'로 수정한다.

(나는 조금 더 깔끔하게 보기위해 위에서 한 replicas도 3으로 줄여주었다.)

수정, 저장하고 나가서 apply -f 해주고 다시 Pod 정보를 확인해보자.

​

docker ps -a | grep nginx 로 디플로이먼트 이름을 먼저 뽑아보고, 다시 버전 확인했다.

​

4) 롤백 기능

kubectl rollout undo deployment [디플로이먼트 이름]

결과: deployment.apps/web-deploy rolled .
