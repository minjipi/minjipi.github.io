0. 삼바(Samba)

삼바(samba)는 SMB(Server Message Block) 또는 CIFS (Common Internet File System)로 알려져 있으며 리눅스(UNIX-like)와 윈도우간에

파일 및 프린터를 공유 할 수 있게 해주는 소프트웨어다.

컴퓨터 간 파일 공유등의 서비스를 구현하는데 SMB/CIFS 프로토콜를 사용하여, UNIX-like, OpenVMS, OS/2, AmigaDOS, 그리고 Netware 등 다양한 플랫폼에서 지원되고

삼바를 통해 리눅스 서버를 타운영체제와 파일을 공유할 수 있는 파일서버로도 사용할 수 있다!

​

즉, 삼바를 이용하면 라즈베리파이의 파일을 윈도우 환경에서 수정하거나 자유롭게 생성할 수 있다는 것.

그럼 라즈베리파이에서 삼바를 구축하고 사용하는 법을 알아보자.

​

1. 라즈베리파이에 삼바 연결하기

(1) 업데이트

$ sudo apt-get update

설치 중 문제가 발생하지 않도록 업데이트를 한다.

​

​

​

(2) 삼바 프로토콜 및 삼바서버, 클라이언트와 관련된 의존성 패키지를 다음의 명령어를 이용하여 설치한다.

$sudo apt-get install samba samba-common-bin

​

​

(3) 삼바 유저 추가, 패스워드 설정

pi@raspberrypi:~ $ sudo smbpasswd -a pi New SMB password: Retype new SMB password: Added user pi. pi@raspberrypi:~ $

pi라는 삼바유저를 추가한다. 자신이 원하는 패스워드를 입력하고

(비밀번호는 원래 보안상 이유로 보이지 않아요)

​

​

​

(4) smb.conf 수정

삼바 설정 파일(smb.conf)을 수정한다. 파일 제일 하단에 새로운 유저 pi 에 대한 설정을 추가한다.

$ sudo nano /etc/samba/smb.conf

​

​

(5)

화살표키로 제일 아래로 내려가서 다음 명령줄들을 입력한다. 입력이 끝나면 ctrl-x (저장), 그리고 y 를 누르고, 같은 파일 이름이므로 enter 를 누르고 빠져나오면 된다.

[pi] comment = rpi samba path = /home/pi valid user = pi writable = yes read only = no browseable = yes

* comment - 삼바 서버에 대한 설명

path - 삼바 서버로 파일을 다룰 디렉토리= 삼바 서버 접속 시 최초 화면

valid user - 사용자

browseable - 공유 폴더의 목록을 보여주는 옵션

​

​

​

(6) 삼바 재실행

​

위 코드를 입력하고, 설정된 내용이 반영될 수 있도록 삼바를 재실행시켜준다.

$ sudo service samba restart 위 코드가 안되면, $sudo /etc/init.d/samba restart

그럼 아래의 문구가 뜨며 재시작 된다.

$ sudo /etc/init.d/samba restart [ ok ] Restarting nmbd (via systemctl): nmbd.service. [ ok ] Restarting smbd (via systemctl): smbd.service. [ ok ] Restarting samba-ad-dc (via systemctl): samba-ad-dc.service.

​

​

(7) 공유폴더접속

윈도우의 탐색기 혹은 네트워크를 열어서 폴더 주소 칸에 \\192.168.1.238\ 과 같이 라즈베리파이의 IP 주소를 적어주면, 공유를 설정해둔 라즈베리 파이 폴더에 접속할 수 있다. 여기에서 새로운 디렉토리를 만들고, 파일의 수정, 삭제, 복사, 이동이 가능하게 된다.
