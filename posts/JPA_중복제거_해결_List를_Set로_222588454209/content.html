[ 상황 ]

게시글1 -> 게시글1 관련 글1 -> 게시글1 관련 글2 -> 게시글 1의 댓글

게시글 1개에 대한 관련글이 2개, 게시글 1개에 대한 댓글이 1개 존재함.

아래 그림이 내가 원하는, 예상 결과 페이지였다.

DB에도 다음과 같이 저장되어 있다.

​

그리고 스프링부트 repository 코드에는 다음과 같이 작성했다.

@Query("SELECT r, ri, s, rc FROM Recipe r LEFT OUTER JOIN RecipeImage ri on ri.recipe = r LEFT OUTER JOIN Story s on s.recipe=r LEFT OUTER JOIN Rcomment rc ON rc.recipe = r WHERE r.idx = :idx") List<Object[]> getRecipeWithImageAndStory(int idx);

​

그런데 결과는 예상과 다르게.. 아래와 같은 문제가 있었다.

게시글 관련 글의 수 만큼 게시글 1의 댓글이 생기는 중복문제.

​

게시글 관련 댓글은 하나 뿐인데,

(ex, "내가 첫번째 댓글 달았다!")

이상하게도 관련글 갯수(2개) 만큼 댓글이 늘어났다.

(ex, "내가 첫번째 댓글 달았다!", "내가 첫번째 댓글 달았다!")

​

[ 문제 ]

repository에 작성한대로 디비 조회를 해보니..

여기서부터 중복 문제가 생겼었다.

​

​

[ 해결 ]

중복 문제가 있었던 이유는 게시글 DTO안에 댓글 DTO가 List 형태였기 때문이었다.

(자바에서 List는 중복이 있고, Set는 중복이 없음)

public class 게시글DTO { int idx; String title; String contents; String writer; @Builder.Default private Set<관련글DTO> 관련글DTOSet = new HashSet<>(); @Builder.Default private Set<댓글DTO> 댓글DTOSet = new HashSet<>(); }

​

그래서 댓글 DTO를 아래와 같이 Set 형태로 수정했다.

private Set<게시글DTO> 게시글DTOSet = new HashSet<>();

고치고 나니 문제가 없어졌다. 해결!
