(5) 컴퓨트 노드에 LVM 설정

[2] compute

dnf -y install lvm2 device-mapper-persistent-data
mkdir /var/lib/cinder
fallocate -l 300G /var/lib/cinder/cinder-volumes
losetup /dev/loop0 /var/lib/cinder/cinder-volumes
vgcreate cinder-volumes /dev/loop0

​

compute

dnf -y install openstack-cinder targetcli python3-keystone

(7) 컴퓨트 노드에 cinder 설정

vi /etc/cinder/cinder.conf
[database] connection = mysql+pymysql://cinder:비번@controller/cinder [DEFAULT] transport_url = rabbit://openstack:비번@controller auth_strategy = keystone my_ip = 컴퓨트노드 IP enabled_backends = lvm glance_api_servers = http://controller:9292 [keystone_authtoken] www_authenticate_uri = http://controller:5000 auth_url = http://controller:5000 memcached_servers = controller:11211 auth_type = password project_domain_name = default user_domain_name = default project_name = service username = cinder password = 비번 [lvm] volume_driver = cinder.volume.drivers.lvm.LVMVolumeDriver volume_group = cinder-volumes target_protocol = iscsi target_helper = lioadm lvm_type = auto [oslo_concurrency] lock_path = /var/lib/cinder/tmp

​

systemctl enable openstack-cinder-volume.service target.service
systemctl start openstack-cinder-volume.service target.service

(8) 컨트롤러 노드에서 확인

openstack volume service list

결과 2개 나오고 up 떠야함.

​

** compute에서 up 안될 때 && cinder Volume driver LVMVolumeDriver not initialized 에러 :

​

컴퓨트 노드에서 /etc/cinder/cinder.conf 파일

[lvm] 부분 마지막에 lvm_type = auto 추가 후, systemctl start openstack-cinder-volume.service

이 부분 오픈스택 메뉴얼 페이지에 나와있지 않아서 에러날 수 있음.
