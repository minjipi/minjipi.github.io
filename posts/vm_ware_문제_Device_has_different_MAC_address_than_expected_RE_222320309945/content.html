vm ware 에러

문제1 : Device has different MAC address than expected, ignoring.

문제2: WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED

​

​

1.

VMWare 또는 VirtualBox에서 리눅스 가상 머신을 복제할 때 간혹 네트워크 카드의 맥주소가 변경되어 꼬이는 경우가 있다. 보통 부팅시 다음과 같은 오류 메세지가 출력된다.

Bringing up interface eth0: Device eth0 has different MAC address than expected, ignoring.

원인은 기존 설정시에 잡힌 mac address 때문인데, 직접 eth의 hwaddr 정보를 수정하면 해결된다.

​

1. 새로운 device의 mac address를 확인

ifconfig -a

​

2. ifcfg-eth0 에서 hwaddr 에 새로운 device의 mac address를 적어준다.

vi /etc/sysconfig/network-scripts/ifcfg-eth0

​

3. 네트워크 재시작 또는 시스템 재부팅

service network restart

​

4. 제대로 접속되는지 확인.

+ 추가 문제 발생.

2. REMOTE HOST IDENTIFICATION HAS CHANGED

나는 eth0 파일의 이름만 eth1로 변경해서 뭔가 추가적인 문제가 생겼다.

​

@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@ @ WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED! @ @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@ IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY! Someone could be eavesdropping on you right now (man-in-the-middle attack)! It is also possible that a host key has just been changed. The fingerprint for the RSA key sent by the remote host is SHA256:2y1lUM4LlGDN/vtmDDkYGcfBGEz/qnXaGCYXA02rqN4. Please contact your system administrator. Add correct host key in /Users/minz/.ssh/known_hosts to get rid of this message. Offending ECDSA key in /Users/minz/.ssh/known_hosts:6 RSA host key for 192.168.100.100 has changed and you have requested strict checking. Host key verification failed.

​

이것의 해결법은 이렇다.

아래 명령어를 입력해주자.

ssh-keygen -R [ IP or DomainName]

그후에 다시 접속하면 이번엔 메시지가 바뀐다.

Are you sure you want to continue connecting (yes/no)?

대략 이런 메시지를 띄운다. yes 입력.

​

위 상황을 예를 들면...

A host 가 있고 B server가 있다.

A는 항상 B 서버에 ssh접속하고 있었는데 B서버에 ssh나 os를 새로 설치 하는 작업을 했다.

그런후에 A는 똑같이 B에 접속을 한다. 이때 B에 IP는 똑같다면, 위와 같은 메시지가 뜬다!

​

이렇게 되는 이유는 단순하다.

ssh 최초접속시에 A와 B에서 서로간에 인증 과정을 하는데.. B는 새로 설치되었으니 B는 상관없지만

A는 예전B에 IP로 인증이 되어있는 상태에서 B로 로그인을 하면

로그인시에 예전에 IP로 인증했던 정보를 가지고 B로 로그인을 하려고 하지만 B는 인증정보가 없기때문에

위와 같은 현상이 나타난다.

​

만약 위에 명령어가 먹히지 않는다면,

find / -name known_hosts 을 통해 해당파일을 찾아 지우면 된다.

보통 /root/.ssh/known_hosts 에 있다.

혹은 /home/username/.ssh/known_hosts

​

(자신의 root면 root에 있는 것을 지워야 되고 일반 유저 이면 /home/username에 있는 걸 지워야함.)
