AWS EC2에 대하여

Elastic cloud com

puting.

​

Vmware의 가상머신을 아십니까?

대충 우리가 가상머신을 만들 때 선택했던 사항들을

Aws의 ec2는 인터넷으로 사용자들이 입력하게 하는거라고 생각하면 된다.

거기에 운영체제도 설치해주고 ssh으로 접속도 할 수 있게 해주는 건데

확인 버튼만 누르면 가상머신을 하나 만들어 주는 편리한 서비스져.

즉, 원격지에 있는 가상머신!

​

RDS란?

AWS RDS에 대하여

똑같이 가상머신을 하나 만들어 달라고 하는건데,

거기에 db서버도 설치해 주는 것.

​

VPC란?

Aws 많은 서비스들이 이런 가상의 네트워크에 생성된다.

중요한건 ec2같은 서비스가 특이하게 ip를 2개 할당받는 점.

​

private ip랑 public ip

private ip는 aws의 vpc 내에서 쓰는 내부ip이며,

Nat와 비슷한 개념.

외부에서는 저 ip로 통신할 수 없다.

그래서 인터넷이랑 연결 해주고 인터넷을 통해 사용자들이 접속 가능하도록 공인 ip 같은 걸 만들어 주는데 그게 바로 public ip.

​

EC2에 ssh접속해서 ifconfig 치면 private ip가 나온다.

그래서 DB랑 연결을 해주려면 같은 vpc안에서 통신하는 거니까

저 private ip를 써줘야 하는 것~~

​

그러면 저 ip만 허용해줬으니까

그 ec2에서만 db로 접속이 가능한것이 된다.

​

보통 DB는 보안상 어디서나 접속할 수 있게 하면 안된다.

Db에서 데이터를 조회하는 웹서버랑 관리자 pc 정도만 접속할 수 있게 해준다.

​

그래서 mysql이라고 하는 프로그램도, 서버 프로그램와 클라이언트 프로그램이 따로 있다.

일반적으로 설치하고 실행해서 포크3306을 리스닝 중인데 서버 프로그램이고

클라이언트 프로그램은

mysql -u root -p

이렇게 명령어로 쓰던 저 mysql이라는 프로그램

저게 클라이언트 프로그램 중 하나이고 또 하나는 mysql workbench.

* Putty도 클라이언트 프로그램이긴한데 MySQL의 클라이언트가 아니라 텔넷 서버

Ssh서버 이런 서버들의 클라이언트 프로그램.

​

그래서 본체에서 워크벤치로 접속하는 것보다 Ec2에서 접속하는게 더 편하니까 Ec2의 ip를 허용해주라 한 것.

Ip중에서도 private ip!

​

그럼 데이터 베이스도 생성하고 감도 잡았으니

ec2에 mysql 클라이언트 프로그램을 설치하고 접속해보자..
