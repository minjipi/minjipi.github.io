근데 이 transferTo 메서드 내부를 보면

public long transferTo(OutputStream out) throws IOException { Objects.requireNonNull(out, "out"); long transferred = 0; byte[] buffer = new byte[DEFAULT_BUFFER_SIZE]; int read; while ((read = this.read(buffer, 0, DEFAULT_BUFFER_SIZE)) >= 0) { out.write(buffer, 0, read); transferred += read; } return transferred; }

1을 구현해둔 느낌.

​

Java 에서 inputStream.transferTo 사용 시, 기존 inputStream 의 데이터는 날아가나?

Java의 InputStream.transferTo() 메서드는 주어진 입력 스트림의 데이터를 다른 출력 스트림으로 전송한다.

​

transferTo() 메서드는 내부적으로 버퍼를 사용하여 데이터를 전송하며, 데이터를 읽으면서 동시에 출력 스트림으로 쓰게 된다. 따라서 전송이 진행되면 입력 스트림의 데이터는 버퍼에서 소비 후 뒤로 밀려나서, 마치 소멸되는듯 보인다. 해당 부분을 다시 읽으려면 따로 mark, reset 등의 메서드를 사용해야한다.

​

그래서 만약 기존의 입력 스트림의 데이터를 보존하고 싶다면, transferTo() 메서드를 호출하기 전에 기존의 입력 스트림을 복사하여 새로운 입력 스트림을 생성해야 한다.
