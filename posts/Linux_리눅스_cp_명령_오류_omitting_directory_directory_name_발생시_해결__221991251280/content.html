Linux 에서 cp 명령시 오류 (omitting directory 'directory name') 발생시

해결 방법

현상이 발생하는 이유:

​

- 해당 디렉토리가 사용중일 경우

- 하위 디렉토리는 복사를 하지 않을때

​

​

아래와 같이 -r 옵션을 사용하여 해결이 가능하다.

cp -r [원본 디렉토리] [복사할 위치]

​

​

cp -r 옵션의 의미

-r 옵션은,

만약 a가 디렉토리라면 그 하위에 디텍토리도 모두 복사한다 는 의미.

​

ex)

cp -r /a /b

: a라는 파일이나 디렉토리를 b라는 이름으로 복사 (b가 없으면 새로 생성)
