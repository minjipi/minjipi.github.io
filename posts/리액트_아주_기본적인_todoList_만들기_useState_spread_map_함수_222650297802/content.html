2. map 함수

자바스크립트 반복문을 통해 DOM에 JSX*을 렌더링하고 싶을 때, return 안에서 for을 쓸 수 없다.

* JSX(JavaScript XML)는 Javascript에 XML을 추가한 확장한 문법.

JSX는 리액트로 프로젝트를 개발할 때 사용되므로, 공식적인 자바스크립트 문법은 아니다.

​

대체 방법은 두가지가 있는데,

​

1) return문에서 JSX를 반환하는 함수 호출 하는 것.

function App() { const num = ["1", "2", "3", "4"]; const rendering = () => { const result = []; for (let i = 0; i < num.length; i++) { result.push(<span key={i}>{num[i] + " / "}</span>); } return result; }; return <div>{rendering()}</div>; }

장점은 재사용이 용이하며 return문이 깔끔하다는 것이다.

​

2. return문 안에서 map 함수 사용.

function App() { const num = ["1", "2", "3", "4"]; return ( <div> {num.map((number, index) => ( <span key={number}> {number} {" / "} </span> ))} </div> ); }

장점은 가독성이 좋고 직관적이라는 점.

forEach는 값을 반환하지 않는 단순연산이기 때문에 JSX를 반환할 수 있는 map함수를 사용한다.

나는 map함수를 사용했다.

todoList && todoList.map((todo) => <p>{todo}</p>)

todoList가 있을 때,

todoList의 값을 한 씩 꺼내서 todo 라는 이름으로 => {} 이 실행문에서 수행한다.
