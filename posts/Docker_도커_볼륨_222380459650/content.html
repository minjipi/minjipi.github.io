# 도커 볼륨 확인

간단한 실습으로, centos7과 mysql 컨테이너를 생성해 둘의 볼륨을 확인해 본다.

docker run -d --name centos7 centos:7 bash
docker run -e MYSQL_ROOT_PASSWORD=qwer1234 -d --name mysql mysql:latest

​

docker inspect centos7 | grep vol docker inspect mysql | grep vol

2) 볼륨 활용 방법

컨테이너 데이터를 영구적으로 보관 가능

(1) 호스트 볼륨 공유

호스트와 저장장소를 공유.

docker run -it --name centos1 -v /vol:/vol centos:7 bash

​

# 실습

docker run -it -p 8081:8080 --name test1 -v /vol:/usr/local/tomcat/webapps/minji -d tomcat
docker exec -it test1 bash

​

(2) 볼륨 컨테이너

데이터 저장용 컨테이너.

-v 옵션으로 볼륨을 사용하는 컨테이너를 다른 컨테이너와 공유.

docker run -it --name centos2 -volumes-from centos1 centos:7 bash

컨테이너 생성시 --volumes-from 옵션을 사용하면 -v 옵션이 적용된 컨테이너의 볼륨 디렉토리를 공유할 수 있다.

모든 데이터가 이 볼륨 컨테이너에 저장되는 것.

​

(3) 도커가 관리하는 볼륨

docker volume 명령어를 이용하여, 도커 자체에서 제공하는 볼륨 기능을 활용해 데이터를 보존 가능.

​

​

볼륨 생성

docker volume create --name my_vol

​

centos3 생성

docker run -it --name centos3 -v mt_vol:/root/ centos:7 bash

​

​

* * stateless vs stateful :

stateless : 컨테이너가 아닌 외부에 데이터를 저장하고 컨테이너는 그 데이터로 동작하도록 설계 방식.

stateful : 컨테이너가 데이터를 저장하는 설계방식.

​

​
