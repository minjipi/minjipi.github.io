AWS EC2서버에 리액트 배포를 하고 싶었는데, 자꾸 에러나서 일단 그냥 실행으로 띄워봤다.

큰 차이 없을거라 생각했지만 파일 용량이 예상보다 커서 수정할 예정이다.

이렇게 하는 사람 별로 없겠지만 혹시 나처럼 그냥 함 돌려보실 분들을 위해 정리한다.

​

​

AWS EC2서버에 React 앱 실행하기

​

  1. 프로젝트 경로에서 압축

tar cvf web.tar vanilla-front (프로젝트 이름)

​

2. aws ec2서버로 파일 옮기기

scp -i /Users/key.pem web.tar ubuntu@12.345.456.78:/home/ubuntu/web.tar

​

3. aws ec2서버 접속

ssh -i /Users/key.pem ubuntu@12.345.456.78
sudo su - root

​

4. Node 14.x 버전 install

​

curl -sL https://deb.nodesource.com/setup_14.x | sudo -E bash -
sudo apt-get install -y nodejs

​

설치확인

node -v npm version

버전이 14.x로 잘 표기된다면 설치 성공~

​

5. aws ec2서버에서 파일 압축 풀기

cd /저장된 경로 tar xvf web.tar

​

6. ls 명령어로 압축 풀린 것 확인하고, 압축 풀린 폴더로 들어가서 다음과 같이 port 80 으로 변경 후 실행.

(변경 안하면 유저가 주소창에 도메인:3000 이렇게 포트번호까지 입력하고 접속해야함)

PORT=80 npm run start &

&은 백그라운드로 실행 옵션!

​

​

* 혹시 이미 사용 중인 포트라거나.. 에러 났을 땐 아래 명령어로 확인.

ps -ef | grep [PID 혹은 프로그램 이름]

​

이미 사용중인 포트 있으면 kill로 꺼주기(중요하지 않은 프로그램일 경우)

​

​

+ AWS EC2의 보안그룹 인바운드 규칙에 HTTP(80) 혹은 필요한 포트 허용해주기.

인바운드 규칙은 외부에서 서버로 네트워크 접근할 때 허용할 포트를 설정하는 규칙으로 웹 서버 구축 시 필수적으로 필요하다.
