[스트레티지 패턴으로 코드 수정 1]

interface State { public String on_button_pushed(Light light); public String off_button_pushed(Light light); }

​

interface State 에서

State On과 Off 버튼에 대한 동작을 정의 하는 코드를 정의한다.

​

public String on_button_pushed (Light light);

라이트의 바뀐 상태를 설정해 줘야 하니까

매개 변수로 light 객체를 받는다.

class ON implements State { @Override public String on_button_pushed(Light light) { System.out.println("반응없음"); return ImageService.lightImageName[0]; } @Override public String off_button_pushed(Light light) { light.setState(new OFF()); System.out.println("불 꺼요"); return ImageService.lightImageName[1]; } } class OFF implements State { @Override public String on_button_pushed(Light light) { light.setState(new ON()); System.out.println("불 켜요"); return ImageService.lightImageName[0]; } @Override public String off_button_pushed(Light light) { System.out.println("반응없음"); return ImageService.lightImageName[1]; } }

​

이전에는 if문으로 복잡하게 코드를 구현했었는데, 바뀐 코드를 통해서

내 상태가 On일때 on button이 눌리게 되면,

변경될 것이 업다는 것을 바로 띄워줄 수 있게 되었다.

​

-> 객체 자체가 처리를 하기 때문에 현재내 상태에 대해서는

if문으로 내 상태를 판단할 필요가 없어진다는 것이 확인 가능하다💡

​

State 쪽은 전혀 손을 댈 필요가 없고..

위와 같이 무드등 상태로 진입하는 부분(다음 상태로 변경되는 부분)의 코드만 수정했다.

​

그런데 지금 상태로 변경 할 때마다 new로 객체를 만들어 내고 있다.

메모리를 할당하고 있는 상황..

이 이부분은 싱글턴 패턴을 사용해 조금 더 수정해 주면 좋다.

​

​

싱글턴 패턴 적용

class ON implements State { private static ON on; private ON() { } public static ON getInstance() { if (on == null) { on = new ON(); } return on; } @Override public String on_button_pushed(Light light) { light.setState(SLEEPING.getInstance()); System.out.println("쿨쿠울"); return ImageService.lightImageName[2]; } @Override public String off_button_pushed(Light light) { light.setState(OFF.getInstance()); System.out.println("불 꺼요"); return ImageService.lightImageName[1]; } }

​

스테이트 패턴, 스트레티지 패턴 차이점

스트레티지 패턴은 전략을 수정하기 위해 사용된다.

예를들어 게임 캐릭터 각각의 설정들,

다른 알고리즘으로 손쉽게 교체하는 동작에 자주 쓰인다.

​

이와 조금 다르게 스테이트 패턴은,

상태에 따라 다양한 행동을 처리할 때 사용한다.

행위 자체도 상태 객체에 위임을 해서 처리하고 있다.
