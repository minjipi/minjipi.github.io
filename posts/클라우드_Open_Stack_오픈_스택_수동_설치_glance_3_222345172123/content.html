2. glance (이미지 서비스)

(1) DB 생성

[1] controller

mysql -u root -p
CREATE DATABASE glance;
GRANT ALL PRIVILEGES ON glance.* TO 'glance'@'localhost' IDENTIFIED BY '암호'; GRANT ALL PRIVILEGES ON glance.* TO 'glance'@'%' IDENTIFIED BY '암호';

​

(2) 키스톤에 이미지 서비스 추가

[1] controller

source ~/keystone_admin
openstack user create --domain default --password-prompt glance
openstack project create --domain Default --description "Service Project" service
openstack role add --project service --user glance admin

​

만약 이 명령어를 먼저 입력할 경우, 다음과 같은 에러가 난다.

No project with a name or ID of 'service' exists.

openstack role 명령어는 관리자한테 역할을 추가 해주는 것이다.

거기 명령어에 프로젝트를 지정하는 옵션이 있는데, 그 프로젝트 이름이 service인 것.

그래서 위 명령어를 가장 먼저 입력할 경우

service 라는 프로젝트가 만들어진게 앖어서 찾을 수 없다고 에러가 나는 것이다. (매뉴얼에는 해당 프로젝트를 생성하는 명령어가 정리되어 있지 않아서 설명..)

​

서비스추가)

openstack service create --name glance --description "OpenStack Image" image
openstack endpoint create --region RegionOne image public http://controller:9292 openstack endpoint create --region RegionOne image internal http://controller:9292 openstack endpoint create --region RegionOne image admin http://controller:9292

(3) 서비스 설치

[1] controller

dnf -y install openstack-glance

​

(4) 서비스 설정

[1] controller

vi /etc/glance/glance-api.conf

​

[database] 한 줄 아래에 추가

connection = mysql+pymysql://glance:암호@controller/glance

​

[keystone_authtoken] 한 줄 아래에 추가

www_authenticate_uri = http://controller:5000 auth_url = http://controller:5000 memcached_servers = controller:11211 auth_type = password project_domain_name = Default user_domain_name = Default project_name = service username = glance password = 암호

​

[paste_deploy] 한 줄 아래에 추가

flavor = keystone

​

[glance_store] 한 줄 아래에 추가

stores = file,http default_store = file filesystem_store_datadir = /var/lib/glance/images/

​

su -s /bin/sh -c "glance-manage db_sync" glance
systemctl enable openstack-glance-api.service
systemctl start openstack-glance-api.service
