화살표 함수 (Arrow Function)

ES6 문법 화살표 함수는 function 키워드를 사용해서 함수를 만든 것보다 간단히 함수를 표현할 수 있다.

또 콜백함수에서 this를 bind 해야하는 문제도 발생하지 않는다.

일반 함수와의 차이를 살펴보자.

​

[일반 함수]

[1번 예시] function oldFunction() { } [2번 예시] var oldFunction = function(){ // 실행문 }

함수는, 1번이나 2번처럼 만들고,

oldFunction(); 이렇게 함수를 사용한다.(다른 방법도 있지만)

그런데 화살표 함수를 사용하면 이렇게 만들 수 있다.

​

​

[ 화살표 함수 ]

newFunction = () => {'나는 화살표 함수!'};

꽤 간결하고 직관적인 문장.

​

​

화살표 함수의 기본문법

함수이름 = (매개변수) => {표현식};

​

1. 매개변수가 없는 경우

var arrow = () => console.log('화살표'); arrow();

​

2.매개변수가 하나인 경우

var arrow = x => x; arrow('화살표');

​

3. 매개변수가 여려개인 경우

var arrow = (a, b) => a + b; // 간단하게 한줄로 표현할 땐 "{}" 생략가능! arrow(1, 2);
var arrow = (a, b) => { return a + b }; arrow(1, 2);

​

​

그리고 화살표 함수를 사용하면

콜백함수에서 this를 bind 해야하는 문제도 발생하지 않는다. * https://velog.io/@colki/JS-this-%EB%B0%94%EC%9D%B8%EB%94%A9-%EC%BD%9C%EB%B0%B1%ED%95%A8%EC%88%98%EC%97%90%EC%84%9C%EC%9D%98-this

즉 내부에서 this 값을 쓸 때, 밖에 있던 this 값을 그대로 사용한다.

화살표 함수는 어디서 쓰든간에 내부의 this 값을 변화시키지 않는다.

그러니까 바깥에 있던 this의 의미를 그대로 내부에서도 사용하는 함수인 것이다.
