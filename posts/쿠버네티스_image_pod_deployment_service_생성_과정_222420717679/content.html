# 전체 과정

1. 스프링부트에서 .jar 파일 생성

FROM openjdk:8u292-slim-buster RUN apt-get update RUN apt-get -y install curl WORKDIR /app ENV CONFIG_SERVER_IP localhost ENV EUREKA_SERVER_IP localhost COPY .jar /app/ ENTRYPOINT ["java", "-jar", ".jar"]

​

2. 이미지 생성

docker build -t ghdalswl77/spring-cloud-eureka-server:1.0 ./ docker build -t ghdalswl77/spring-cloud-config-server:1.0 ./ docker build -t ghdalswl77/spring-cloud-zuul-server:1.0 ./ docker build -t ghdalswl77/msa-service-coffee-order:1.0 ./ docker build -t ghdalswl77/msa-service-coffee-status:1.0 ./
docker build -t ghdalswl77/

​

3. 도커 허브에 이미지 업로드

docker push ghdalswl77/msa-service-coffee-member:1.0

​

4. worker 2에서 이미지 다운

* 컨테이너 만들어지는 곳에서 미리 image pull 해주면 더 빨리 생성된다.

docker pull ghdalswl77/spring-cloud-config-server:1.0

​

5. 서비스 생성

svc-config-server.yaml apiVersion: v1 kind: Service metadata: name: svc-config-server spec: selector: app: config-server ports: - protocol: TCP name: config-server-port port: 8888

설정 서버는 외부 연결 필요 없으니, 따로 지정해 주지 않고 Cluster IP.

유레카, 주울 서버는 NodePort 지정을 해준다.

​

​

6. 이미지로 디플로이먼트(파드) 생성

​

# 💡deployment 구조

아래 구성도를 참고해 deployment 작성.

​

(1) deploy-config-server.yaml

유레카 아이피 환경변수로 넘겨줌.

설정 서버는 유레카와 연결 되어 있어야 하므로!

​

(2) deploy-eureka-server

유레카는 설정 서버 아이피, 유레카 서버 아이피 환경 변수로 넘겨줌.

​

​

(3) deploy-zuul-server.yaml

​

​

(4,5) deploy-coffee-status.yaml, deploy-coffee-order.yaml

유레카 아이피, 카프카 아이피, 카프카 포트

# 이미지를 지울 땐

- 도커 허브에 올라간, 이미지 삭제

- 파드가 생선된 워커에서, 이미지 삭제

- 마스터에서, kubectl delete -f deploy-coffee-order.yaml 로 디플로이 먼저 지우고, 이미지 삭제.
