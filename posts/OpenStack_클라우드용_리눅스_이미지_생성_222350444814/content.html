< ISO 파일로 클라우드용 이미지 만들기 요약 >

​

2. 가상머신에 운영체제 설치

​

3. 부팅시 DHCP로 네트워크 설정이 되도록 설정

​

4. acpid, cloud-init, cloud-utils-growpart

​

5. 추가적인 설정하기 (zeroconf 설정, 부팅 환경설정)

​

6. 확인 (오픈스택에서 이미지 등록하고 가상머신 생성, QEMU + KVM로 수동으로 가상머신 생성)

​

실습 환경(이미지 생성) 준비

vmware같은 하이퍼바이저 종류인

QEMU & KVM 설치.

dnf module install virt dnf install virt-manager virt-install virt-viewer libguestfs-tools qemu-img

​

CentOS7 ISO 파일 다운로드.

wget https://mirror.umd.edu/centos/7/isos/x86_64/CentOS-7-x86_64-Minimal-2009.iso

​

(0) qcow2 이미지(디스크) 파일 생성

가상의 하드디스크(비어있는 하드디스크)를 생성하는 것.

qemu-img create -f qcow2 [이미지 파일 위치] [이미지 파일 크기] qemu-img create -f qcow2 /virt/centos7.qcow2 20G

이것으로 가상 머신을(centOS7를 설치할 가상머신을) 생성!

​

** VMware 에서는 이 작업 시 splite file로 vmdk 파일이 나누어 저장되는데,

(손상 시, 피해 최소화를 위해) 여기서는 centos7.qcow 파일이 하나인 듯 single file로 저장된다.

그리고 VMware에서도 single file 만들 수 있음!

​

(1) 가상 네트워크 생성

virsh net-start default

default 라는 이름의 가상 네트워크 생성.

​

virsh net-list

생성한 가상 네트워크 확인.

​

(2) 가상 머신 생성

virt-install --name centos7 --ram 1024 \ [생성할 가상머신 이름] [생성할 가상머신 메모리] --disk /virt/centos7.qcow2,format=qcow2 \ [가상머신 하드디스크의 경로] --network network=default \ [가상머신이 쓸 네트워크] --graphics vnc,listen=0.0.0.0 --noautoconsole \ [vnc설정. 가상머신 화면 받아오는..] --os-type=linux --os-variant=centos7.0 \ [운영체제 타입과 종류] ** 생성 가능한 가상머신 목록 확인은 'osinfo-query os'로 확인. --location=/virt/CentOS-7-x86_64-Minimal-2009.iso [운영체제 설치 ISO 이미지 경로]
virt-install --name centos7 --ram 1024 \ --disk /virt/centos7.qcow2,format=qcow2 \ --network network=default \ --graphics vnc,listen=0.0.0.0 --noautoconsole \ --os-type=linux --os-variant=centos7.0 \ --location=/virt/CentOS-7-x86_64-Minimal-2009.iso

(3) 운영체제 설치 진행

Reboot

​

(4) 클라우드용 이미지로 만들기 작업

[0] 네트워크 설정

​

먼저 네트워크 설정

dhclient

자동으로 아이피 할당받고,

​

​

[1] acpid 설치 및 자동 실행 설정

acpid 설치.

하이퍼바이저가 가상머신을 재부팅 또는 종료할 수 있게 해주는 프로그램!

yum install acpid

​

자동 실행 설정

systemctl enable acpid

​

[2] cloud-init 설치 및 설정

cloud-init : 가상머신이 생성될 때 초기화 되어야 하는 호스트 이름, 네트워크 인터페이스, 그리고 인증키를 설정하는 것과 같은 가상 머신 초기 설정을 자동화하는 도구.

(여기서는 ssh 공개키를 가상머신에 넣도록 해줄 것임)

​

cloud-init 설치)

yum install cloud-init

설정파일 위치 : /etc/cloud/cloud.cfg

​

​

[3] cloud-utils-growpart

하이퍼바이저가 가상머신의 하드디스크 용량을 늘렸다- 줄였다 할 수 있게 해주는 프로그램.

yum install cloud-utils-growpart

* lvm resize2fs 명령어 처럼, 하드디스크 용량을 자동으로 관리 해주는 것.

​

[4] zeroconf 설정

가상머신이 메타데이터 베이스에 접근할 수 있도록 하는 설정.

* 메타데이터 베이스 : 오픈스택이 가상머신에 대한 정보(아이피, 에러 등..)

echo "NOZEROCONF=yes" >> /etc/sysconfig/network

​

[5] nova에게 로그를 보여줄 수 있도록 설정

vi /etc/default/grub

들어가서 rhgb quiet 부분을 console=tty0 console=ttyS0,115200n8 로 변경.

grub2-mkconfig -o /boot/grub2/grub.cfg

​

[6] 전원 종료

poweroff

​

[7] 호스트PC에서 다음을 설정

virt-sysprep -d [가상머신이름]

위 명령어로 하드디스크에 기록된 고유값(맥주소 등..)을 삭제한다.

왜냐면 우리는 이 것으로 여러대의 컴퓨터를 생성할 것이라 이것을 지워주고 사용!
