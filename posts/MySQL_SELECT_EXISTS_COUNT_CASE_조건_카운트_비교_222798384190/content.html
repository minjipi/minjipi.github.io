1. SELECT EXISTS

두 테이블 간의 결과를 어떤 값이 존재하는 지를 알고 싶은 경우,

새로운 데이터를 입력할 때 해당 데이터가 이미 존재하는지 확인할 때도 유용하다.

핵심은 '존재하는가'를 판단하는 것.

​

[두 테이블 간의 결과를 어떤 값이 존재하는 지 확인]

SQL에서 연관 서브쿼리 연산자 EXISTS 활용하기

​

[새로운 데이터를 입력할 때 해당 데이터가 이미 존재하는지 확인

mysql> SELECT EXISTS(SELECT * FROM member WHERE email = 'minji'); +---------------------------------------------------+ | EXISTS(SELECT * FROM member WHERE email = 'minji') | +---------------------------------------------------+ | 1 | +---------------------------------------------------+ 1 row in set (0.00 sec)

​

2. COUNT CASE (조건 카운트)

테이블의 Row 갯수를 구할때 사용.

특정 조건에 해당하는 행의 카운트를 조건에 일치할 경우 카운트하고, 일치하지 않을 경우 카운트하지 않는다.

SELECT COUNT(CASE WHEN 조건 THEN 1 END) FROM 테이블

​

[예시]

SELECT COUNT(CASE WHEN idx = ? and status = 1 THEN 1 END) as rowN from member"

​

​

🤔 select count vs if exists

(특정조건의 자료가 있는지 확인만 하면 되는 경우)

​

약 200만건정도의 더미데이터가 있는 테이블로 확인해본 결과, EXISTS를 사용했을때 약 100배정도 성능향상이 있었다. EXISTS는 서브쿼리내에서 WHERE조건절을 만족하는 행이 하나라도 발견되면 스캔을 멈추고 TRUE를 반환하게 된다. 따라서 TBL_BOARD의 건수가 아무리 많더라도 비교적 빠르게 존재유무를 판단할 수 있다.

​

EXISTS : true로 판명되는 즉시 중지할 수 있지만 COUNT(*)는 한 행이 테스트를 통과했음을 알고도 모든 행을 계산해야 한다.

COUNT : 예를들어 한번만 주문을 했던 회원은 한건을 count하겠지만 1000번 주문을 한 회원의 경우

1000건 모두를 읽어서 count를 하게된다.
