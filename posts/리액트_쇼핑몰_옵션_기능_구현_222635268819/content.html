5. '옵션의 옵션' 선택하면 옵션에 선택한 값 전달되면서 옵션 창 닫히게

const [selectedVal, setSelectedVal] = useState(""); const [isSelectVisible, setIsSelectVisible] = useState(false);

isSelectVisible, selectedVal 이라는 스테이트를 만든다.

selectedVal은 선택된 옵션 값을 저장하고,

isSelectVisible은 선택된 옵션이나 옵션의 옵션을 보여줄지 말지 상태를 저장한다.

* props 와 state.

props는 부모 컴포넌트가 -> 자식 컴포넌트에게 주는 값.

자식 컴포넌트에서는 props를 받아오기만하고, 받아온 props를 직접 수정 할 수 는 없다.

state는 컴포넌트 내부에서 선언하며 내부에서 값을 변경 할 수 있다.

props는 (함수 매개변수처럼) 컴포넌트에 전달.

state는 (함수 내에 선언된 변수처럼) 컴포넌트 안에서 관리된다는 차이!

props를 사용했는데도 state를 사용하는 이유:

사용하는 쪽과 구현하는 쪽을 철저하게 분리시켜서 양쪽의 편의성을 각자 도모하는 뭐 그런

​

import React, { useState } from "react"; import styled, { css } from "styled-components"; function Option(props) { const [isSelectVisible, setIsSelectVisible] = useState(false); const [selectedVal, setSelectedVal] = useState(""); return ( <ol className={isSelectVisible ? "active" : ""}> <li> <span>{props.optionData.title}</span> <span>{selectedVal}</span> <i onClick={() => { setIsSelectVisible(!isSelectVisible); }} className={ isSelectVisible ? "fas fa-chevron-up" : "fas fa-chevron-down" } /> </li> <div></div> {isSelectVisible ? ( <li className={isSelectVisible ? "active" : ""}> {props.optionData.select.map((select) => { return ( <li> <span onClick={() => { setSelectedVal(select); setIsSelectVisible(!isSelectVisible); }} > {select} </span> </li> ); })} </li> ) : ( "" )} </ol> ); } export default Option;
