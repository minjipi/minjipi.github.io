[ 해결 방법 1 ]

인증서 없이 하는 방법.

정말 간단하지만, 야매인 것 같다..

​

https 방식을 사용하지 않겠다 라고 설정하면 끝이기 때문이다.

​

​

[ 해결 방법 2 ]

인증서를 생성/설정하는 방법이다.

​

(1) Root CA 인증서 생성

명령어 설치

curl -s -L -o /bin/cfssl https://pkg.cfssl.org/R1.2/cfssl_linux-amd64 curl -s -L -o /bin/cfssljson https://pkg.cfssl.org/R1.2/cfssljson_linux-amd64 curl -s -L -o /bin/cfssl-certinfo https://pkg.cfssl.org/R1.2/cfssl-certinfo_linux-amd64 chmod +x /bin/cfssl*
mkdir ~/auth cd ~/auth

디렉토리를 따로 만들어서 json 파일 두 개를 생성한다.

​

vi rootCA-config.json
{ "signing": { "default": { "expiry": "8760h" }, "profiles": { "root-ca": { "usages": ["signing", "key encipherment", "server auth", "client auth"], "expiry": "8760h" } } } }

​

vi rootCA-csr.json
{ "CN": "rootCA", "key": { "algo": "rsa", "size": 2048 }, "names": [ { "O": "Kubernetes" } ] }

​

인증서 생성 :

cfssl gencert -initca rootCA-csr.json | cfssljson -bare rootCA

생성된 것 확인.

​

만든 파일들을 etc/kubernetes/pki/ 에 붙여넣기한다.

(etc/kubernetes/pki/는 쿠버네티스 인증서 파일)

cp ./* /etc/kubernetes/pki/

​

​

(2) 클라이언트용 인증서 생성

관리자의 인증서 생성.

cfssl gencert -ca=rootCA.pem -ca-key=rootCA-key.pem -config=rootCA-config.json -profile=root-ca proxy-client-csr.json | cfssljson -bare proxy-client

​

다시 쿠버네티스 인증서 파일로 옮겨준다.

cp ./proxy* /etc/kubernetes/pki/

​

​

(3) metrics-server 수정 (클라이언트용 인증서 생성)

​

* 저는 방법 1에서 설치한 것을 지워야해서, components.yaml 파일이 있는 곳에서 아래 명령어로 설치를 지워줍니다! 설치가 안된 상태라면 아래 명령어 건너 뛰셔도 됩니다.

kubectl delete -f components.yaml

​

components.yaml 파일 생성

vi components.yaml

​

args: 밑에 쯤

(방법 1에서 해준 - --kubelet-insecure-tls 이 있다면 이 부분 지우고)

아래 4줄 추가.

- --requestheader-allowed-names=front-proxy-client - --requestheader-extra-headers-prefix=X-Remote-Extra- - --requestheader-group-headers=X-Remote-Group - --requestheader-username-headers=X-Remote-User

​

수정한 파일 적용

kubectl apply -f components.yaml

​

​

(4) 확인

kubectl get apiservices | grep metric

파일 적용 후 위 명령어로 확인해본다.

바로는 안되고 1~2분 정도 후에 True 올라옴.
