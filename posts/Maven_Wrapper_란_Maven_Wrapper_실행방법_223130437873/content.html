Maven Wrapper 란?

Maven Wrapper는 Maven Java 기반 프로젝트의 빌드, 의존성 관리, 테스트, 패키징 등을 관리하는 빌드 도구 프로젝트를 실행할 수 있는 도구입니다.

Maven Wrapper는 이 Maven을 설치하지 않고도 Maven 프로젝트를 실행할 수 있도록 도와줍니다.

​

일반적으로 Maven을 사용하려면 로컬 개발 환경에 Maven을 설치해야 하는데,

Maven Wrapper를 사용하면 프로젝트 디렉토리 내에 Maven Wrapper 스크립트와 설정 파일을 함께 포함시켜서

특정 버전을 빌드 할 수 있습니다.

​

ex)

distributionUrl=https://repo.maven.apache.org/maven2/org/apache/maven/apache-maven/3.8.3/apache-maven-3.8.3-bin.zip

프로젝트 내에 다음과 같이 Maven Wrapper 프로젝트 설정 파일을 생성하고, 특정 버전을 명시해줌으로써 빌드 도구의 버전과 설정을 프로젝트와 함께 버전 관리할 수 있습니다.

​

그리고 Maven Wrapper를 사용하면 다른 개발자들이 Maven을 설치하지 않고도 Maven Wrapper 스크립트를 실행하여 프로젝트를 빌드하고 실행할 수 있습니다.

​

​

​

2. Maven Wrapper 실행하기

​

1. .mvn/wrapper 디렉토리 생성

프로젝트 루트 디렉토리에서 .mvn/wrapper 디렉토리를 생성합니다.

​

2. maven-wrapper.properties 파일 추가

.mvn/wrapper 디렉토리 내에 maven-wrapper.properties 파일을 생성합니다. (Maven 3.8.3 버전 예시)

distributionUrl=https://repo.maven.apache.org/maven2/org/apache/maven/apache-maven/3.8.3/apache-maven-3.8.3-bin.zip

원하는 Maven 버전의 배포 URL을 사용하여 해당 버전을 지정할 수 있는데,

Maven 공식 사이트나 Maven 중앙 저장소에서 사용 가능한 버전을 확인하고 URL을 업데이트해야 합니다.

​

​

3. Maven Wrapper 실행

터미널에서 프로젝트 디렉토리로 이동한 후 mvnw 명령어를 실행하여 Maven Wrapper를 실행합니다. (프로젝트 디렉토리에서 mvnw 명령어 실행. 윈도우는 mvnw.cmd)

​

Maven/target Clean 및 build

./mvnw clean package

이때 만 mvnw 실행 권한이 없다면 실행 권한을 추가해야 합니다. (chmod +x mvnw)

​

./mvnw compile
