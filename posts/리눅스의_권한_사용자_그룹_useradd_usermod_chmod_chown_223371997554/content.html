좀 전 /etc/passwd 에서 본 UID와 /etc/group 의 GID는 어떤 차이가 있을까?

​

/etc/passwd의 GID는 각 사용자가 속한 기본 그룹을 식별하는 번호이고,

/etc/group 의 GID는 시스템에 정의된 모든 그룹을 식별하는 번호이다.

​

그래서, 그룹에 가입하는 방식은 2가지가 있다.

​

1번 방식은 한 계정 당 하나의 그룹만 설정할 수 있고, 두번째 /etc/group의 4번째 필드를 사용하는 방법은 한 계정 당 여러개의 그룹에 속할 수 있다는 차이가 있다.

3. 리눅스의 권한

사용자 유형

리눅스에서는 파일 및 디렉터리에 대한 접근 권한을 3가지 사용자 유형으로 나눌 수 있다.

​

a. 소유자(owner): 파일 또는 디렉터리를 생성한 사용자

b. 그룹(group): 파일 또는 디렉터리와 관련된 그룹의 사용자들

c. 기타(other): 소유자와 그룹에 속하지 않은 나머지 사용자들

​

​

권한 유형

각 사용자 유형에 대해 다음과 같은 3가지 권한을 설정할 수 있습니다.

​

읽기(read, r, 4): 파일을 읽을 수 있는 권한

쓰기(write, w, 2): 파일을 수정하거나 삭제할 수 있는 권한

실행(execute, x, 1): 파일을 실행하거나 디렉터리로 이동할 수 있는 권한

​

4. chmod

change mode 파일 권한 변경

chmod 명령어로 파일 권한을 변경하는 방법은 두 가지가 있다.

​

1) Symbolic method (문자 표기법)

ex) 파일 소유자에게 실행 권한 부여

chmod u+x myfile.txt

이 방법은 액세스 클래스(Access class), 연산자(Operator), 액세스 타입(Access Type)으로 구분한다.

액세스 클래스 : u (user), g (group), o (other), a (all)

연산자 : + (add access), - (remove access), = (set exact access)

액세스 타입 : r (read), w (write), x (execute)

명령어 chmod 뒤에 변경할 권한을 입력하면 된다.

​

​

​

2) Absolute form (숫자 표기법)

소유자, 그룹, 다른 사용자의 권한을 r, w, x 순서로 8진수로 해석하여 3자리로 표현한다.

r (read) : 4

w (write) : 2

x (Ececute) : 1

chmod 750 file.txt

7 = r(4) + w(2) + x(1)

5 = r(4) + x(1)

​

​

다음은 리눅스에서 자주 사용되는 권한이다.

​

755: 소유자는 모든 권한을 갖고, 그룹과 기타 사용자는 읽기와 실행 권한만

644: 소유자는 읽기와 쓰기 권한을 갖고, 그룹과 기타 사용자는 읽기 권한만

700: 소유자는 모든 권한을 갖고, 그룹과 기타 사용자는 권한 없음

​

4. chown

change owner 파일 소유권 변경

파일이나 디렉토리의 소유자와 그룹을 변경하는 데 사용

사용자와 그룹은 소유권을 갖는다.

​

ex) 파일 소유자를 user로, 그룹을 group으로 변경

chown user:group myfile.txt

소유자와 소유 그룹을 한번에 지정하는 chown {소유자}:{그룹} 명령어를 자주 사용한다.
