Model의 역할과 생성

​

MVC 패턴은 애플리케이션의 역할을 모델(Model), 뷰(View), 컨트롤러(controller)로 나누어, 이들 사이 상호작용을 통제하는 아키텍처 패턴이다. 이번 포스팅에서는Model의 역할과 생성 방법을 알아본다.

​

Model

MVC패턴의 Model은 애플리케이션의 정보, 데이터를 나타낸다.

데이타베이스, 정보들의 가공을 책임지는 컴포넌트를 말한다.

이 모델은 다음과 같은 특징을 가지고 있다.

​

1. 사용자가 편집하길 원하는 모든 데이터를 가지고 있어야 한다.

2. 뷰나 컨트롤러에 대해서 어떤 정보도 알지 말아야 한다.

3. 변경이 일어나면, 변경 통지에 대한 처리방법을 구현해야만 한다.

​

Controller.java

import org.springframework.stereotype.Controller; import org.springframework.ui.Model; import org.springframework.web.bind.annotation.GetMapping; import org.springframework.web.bind.annotation.RequestMapping; import org.springframework.web.servlet.ModelAndView; @Controller @RequestMapping("/test") public class TestController { @GetMapping("/model") public String model(Model model) { model.addAttribute("testKey", "model Test"); * model Test 라는 문자열 데이터를 testKey라는 키 값에 담아줌. return "/test/model.html"; } @GetMapping("/modelandview") public ModelAndView modelandview() { ModelAndView mav = new ModelAndView(); mav.addObject("testKey", "model and view Test"); * model and view Test를 testKey라는 키 값으로. mav.setViewName("/test/model.html";); return mav; } }

​

모델을 사용하는 방법은 꼭 한가지가 아니다.

위는 model Test 라는 문자열 데이터를 testKey라는 키 값에 담아주었고

아래는 model and view Test를 testKey에 담아주었다.

​

model은 데이터를 담는다는 점이 중요하다. 데이터를 담는 방식의 차이에 대해서는 해당 포스팅에서 중요하지 않게 다루고 넘어간다.(알기로 딱히 없기도 함..)(나는 앞으로 model만 쓸거임~~~~)

​

​

model.html

<!DOCTYPE html> <html lang="ko" xmlns:th="http://www.thymeleaf.org"> <head> <meta charset="UTF-8"> <title>Title</title> </head> <body> <h1 th:text="${testKey}"></h1> </body> </html>
