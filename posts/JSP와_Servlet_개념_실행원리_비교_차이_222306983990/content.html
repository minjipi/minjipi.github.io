JSP와 Servlet 개념, 실행원리, 비교, 차이

​

1. JSP 와 Servlet

JSP = Html 안에, JAVA

Servlet = JAVA 안에, Html

하나는 HTML 안에 JAVA, 또 하나는 JAVA 안에 HTML.

이거나 저거나 같은거 아닌가? 싶은 말인데... 그렇다 !

똑같은 것을 보여주는 코드인데 그 순서만 다르다.

​

​

정말 같은건지, 왼쪽이 Servlet 코드고, 오른쪽은 JSP코드인 이 프로젝트를 실행해 보자.

그럼 아래와 같이 보기에 완전히 똑같은 웹페이지가 띄워진다.

​

코드를 자세히 살펴보도록 하자.

Servlet은 자바 코드 안에 HTML.

​

(JSP도 어쨌든 실행할 때 Servlet로 바뀌어 실행된다.)

​

JSP는 HTML안에 Java 코드가 있다.

이 코드만 보자면, JSP가 당연히 좋은거 아닌가 싶지만 그렇지도 않다.

​

이 코드처럼 간단한 연산에 출력결과가 주가 되는 것은 JSP가 좋고

처리해야할 과정이 복잡한 대신 화면에 출력시키는 것이 없을 경우엔 Servlet이 좋다고 할 수 있겠다.

왜 그런지 자세히 알아보자.

​

2. JSP, Servlet 실행 원리

일단 위 코드와 실행된 페이지를 보면 기능의 차이는 없다는 것을 알 수 있다.

그럼 JSP와 Servlet 이 똑같은걸까?

No!

내부적인 동작과정을 살펴보자.

JSP 먼저 보도록 한다.

Servlet :

​

이 실행 원리를 내가 좋아하는 커피로 예시를 들자면,

내가 콜드브루를 한 잔 마시려고 하는데..

JSP 는 주문과 동시에 콜드브루를 제조하는 방식이고,

​

Servlet 은 내가 마실 콜드브루를 미리 만들어 둬서,

내가 주문하면 바로 마실 수 있는 방식이다.

​

코드를 더 자세히 살펴보자면,

​

JSP는 자바 코드로 바뀌고, 자바 코드를 클래스 파일로 컴파일하고, 컴파일한 클래스로 객체를 생성해서 실행한다.

(실행될 때, 클래스파일로 하나하나 진행)

​

Servlet은 클래스 파일로 미리 컴파일 되어있어서 바로 객체생성해서 실행한다.

3. JSP Servlet 비교

Servlet

JSP

- 웹 기반의 요청에 대한 동적인 처리가 가능한

Server Side에서 돌아가는 Java Program.

​

- Java 코드 안에 HTML 코드 (하나의 클래스).

​

- 웹 개발을 위해 만든 표준.

- Java 언어를 기반으로 하는

Server Side 스크립트 언어.

​

- HTML 코드 안에 Java 코드.

​

- Servlet를 보완하고 기술을 확장한 스크립트 방식 표준Servlet의 모든 기능 + 추가적인 기능.

​

4. JSP Servlet 차이점

JSP :

👉🏻 presentation(View)에 좋다.

즉 요청 결과를 나타내는 HTML 작성하는데 유용하다.

JSP가 수정된 경우 재배포할 필요가 없이 WAS가 알아서 처리한다. (쉬운 배포)

​

Servlet :

👉🏻 data processing(Controller)에 좋다.

Servlet이 수정된 경우 전체 코드를 다시 컴파일한 후 재배포하는 작업이 필요하다. (개발 생산성 저하)

이미 클래스 파일로 컴파일 되어 있어서 수정할 수 없음

​

이 두 가지를 각각 역할에 맞게 사용하면 좋을텐데..

​

​

이 두 가지를 각각 역할에 맞게 사용해서 웹페이지를 만드는 것이, 바로 MVC 패턴이다.

++ 4. MVC 패턴

JSP와 Servlet 모두 이용하는 모델, MVC

MVC 디자인 패턴 :

모델 뷰 컨트롤러(Model-View-Controller, MVC)는 소프트웨어 공학에서 사용되는 소프트웨어 디자인 패턴이다.

​

5. MVC 구성 요소

컨트롤러 C

모델에 명령을 보냄으로써 모델의 상태를 변경할 수 있다.

관련된 뷰에 명령을 보냄으로써 모델의 표시 방법을 바꿀 수 있다.

(서블릿)

사용자로부터 요청을 받고, 그 요청에 해당하는 일을 수행하는 model을 호출

​

모델 M

모델의 상태에 변화가 있을 때 컨트롤러와 뷰에 이를 통보한다.

(클래스)

데이터에 대한 처리, 회원가입을 위해 DB에서 데이터 확인 및 DB에 데이터 기록 후 결과를 컨트롤러에게 알림

​

뷰 V

사용자가 볼 결과물을 생성하기 위해 모델로부터 정보를 얻어 온다.

(jsp)

컨트롤러로부터 처리결과를 JSP 페이지를 통해 보여줌

​
