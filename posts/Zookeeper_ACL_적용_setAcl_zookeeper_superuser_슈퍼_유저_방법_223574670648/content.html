1. 목표

운영 중인 zookeeper 서버 3대에 대해 ACL(Access Control List)을 적용한다.

​

​

​

2. ACL이란?

ACL은 Access Control List의 약자로, Zookeeper의 Znode에 접근할 수 있는 권한을 설정하는 메커니즘이다. 이를 통해 특정 사용자나 IP가 노드에 접근하거나 변경할 수 있는 권한을 부여하여 보안을 강화한다.

​

ACL의 구성 요소

  • Scheme: ACL에서 사용되는 인증 방식

    • world: 모든 사용자에게 동일한 권한을 부여

    • auth: 인증된 모든 사용자에게 권한을 부여

    • digest: 사용자 이름과 비밀번호의 조합으로 인증

    • ip: 특정 IP 주소에 권한을 부여

  • ID: Scheme에 따라 식별되는 사용자 또는 그룹

  • Permission: 특정 Znode에서 허용되는 작업

    • READ: 데이터를 읽을 수 있는 권한

    • WRITE: 데이터를 수정할 수 있는 권한

    • CREATE: 자식 노드를 생성할 수 있는 권한

    • DELETE: Znode 또는 자식 노드를 삭제할 수 있는 권한

    • ADMIN: ACL 설정 및 관리 권한

​

​

​

3. 기존 문제 및 해결 방식

기존에는 방화벽을 사용해 특정 IP만 Zookeeper에 접근할 수 있도록 설정했으나, 이 방식은 환경 특성상 영구적이지 않아서 방화벽 대신 ACL을 적용하여 보안을 강화하기로 했다.

​

참고로.. 일반적으로 Zookeeper는 Kafka 같은 분산 시스템의 코디네이터로 사용되지만, 내가 담당하는 환경에서는 단순 데이터 저장소로만 사용하고 있어서 비교적 간단했다. (이 때문에 Zookeeper를 좀 더 간단하게 다루고 있었으며, 결국 이 서버를 제거했다.)

​

ACL을 적용하는 방식으로는 다양한 선택지가 있었지만 Digest 인증 방식을 사용하기로 결정했다.

Digest 방식은 사용자 이름과 비밀번호를 해시 처리하여 인증하는 방식이다.

​

다른 인증 방식으로는 IP 기반 ACL 방식이 있었으나, 이 방식은 단일 IP에만 권한을 부여할 수 있어 적합하지 않았다.

그리고 SASL (Simple Authentication and Security Layer)을 사용할 수도 있었으나, 쏘심플한 내 환경에 비해 설정이 복잡하고 환경에 맞지 않다고 판단하여 제외했다.

​

​

4. Digest 설정

1) 사용자 생성

Digest 인증을 위해 username:password 형태의 자격증명을 해시한 값을 사용한다.

java -cp ./apache-zookeeper-3.8.4-bin/lib/zookeeper-3.8.4.jar:./apache-zookeeper-3.8.4-bin/lib/slf4j-api-1.7.30.jar org.apache.zookeeper.server.auth.DigestAuthenticationProvider hong:qwer1234

이렇게 사용자를 생성하면, 다음과 같이 hong:qwer1234 에 대한 해시값이 나온다.

​

​

2. Zookeeper 클라이언트에서 인증 수행

클라이언트가 Zookeeper에 접근할 수 있도록 인증 정보를 추가한다.

zkCli.sh -server [Zookeeper IP]:[PORT] > addauth digest hong:qwer1234

​

3. ACL 권한 설정

digest 스킴을 사용하여 인증된 사용자에 대해 특정 Znode에 대한 ACL 권한을 설정한다.

setAcl /moa digest:hong:W24Dg+/UhRuDAraX/MDTt5HQ=:cdrwa

​

* 슈퍼 유저 권한 설정

Acl 작업 시 혹시 계정이 막히면 sudo user 가 필요해서, 슈퍼 유저 권한을 획득해두면 좋다.

슈퍼 유저 권한을 설정하기 위해 java.env 파일에 아래 내용을 추가한다.

SERVER_JVMFLAGS=-Dzookeeper.DigestAuthenticationProvider.superDigest=super:{해시값}

이후, Zookeeper 클라이언트에서 슈퍼 유저로 로그인한다.

addauth digest super:super123

​

ACL이 제대로 설정되었는지 확인하려면 다음 명령어를 사용한다.

getAcl /rootdir

​

5. Spring Boot에서 Zookeeper 클라이언트 설정

Spring Boot 애플리케이션에서 Zookeeper 클라이언트가 ACL을 사용하도록 설정해야 한다.

Spring Cloud Zookeeper 스프링 공식 문서를 참고하자면..

https://docs.spring.io/spring-cloud-zookeeper/reference/config.html

​

Spring Boot에서 Zookeeper 클라이언트로 Digest 인증을 적용하기 위해서 CuratorFramework를 사용하라고 한다.

​

5-1) Curator 클라이언트를 생성할 때 setAuthorization 메서드를 사용하여 Digest 인증을 추가한다.

다음은 Digest 인증을 적용한 CuratorFramework 설정이다.

@Service public class ConfigService { private CuratorFramework client; @Value("${moa.zkConnectionString}") private String zookeeperConnectionString; @Value("${moa.zkNameSpace}") private String zookeeperNameSpace; @PostConstruct public void initService() { client = CuratorFrameworkFactory.builder() .namespace(zookeeperNameSpace) .connectString(zookeeperConnectionString) .retryPolicy(new ExponentialBackoffRetry(1000, 3)) // Digest 인증 추가 .authorization(Collections.singletonList( new AuthInfo("digest", "hong:qwer1234".getBytes()) )) .build(); client.start(); }

​

5-2) 스프링부트에서 ZNode 값 읽기 Spring Boot 애플리케이션에서 ZNode 값에 접근하려면 아래와 같이 CuratorFramework를 통해 ZNode 값을 읽을 수 있다.

public String getZNodeData(String path) throws Exception { return new String(client.getData().forPath(path)); }
