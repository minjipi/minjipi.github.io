2. CREATE TABLE

간단한 사용예제는 다음과 같다:

﻿ CREATE TABLE testTable( (1)id INT(11) NOT NULL AUTO_INCREMENT, (2) name VARCHAR(20) NOT NULL, (3) ouccupation VARCHAR(20) NULL, (4) height SMALLINT, (5) profile TEXT NULL, (6) date DATETIME, (7) CONSTRAINT testTable_PK PRIMARY KEY(id) (8) );

(1) 테이블을 생성하는 명령어.

(2) id라는 컬럼을 추가하는 데, INT 타입으로 지정. NOT NULL은 값을 비워둘 수 없음을 의미한다. AUTO_INCREMENT는 자동으로 값이 1씩 증가하도록 설정하는 옵션.

(3) name이라는 컬럼을 추가하는데, 가변길이로 20글자까지 허용. (20글자가 넘어가면 20글자에서 자른다.)

(4) 위와 동일하지만, 값을 비워두는 것을 허용.

(5) SMALLINT는 INT보다 가질 수 있는 값의 범위가 작다. 메모리 측면에서 이득!

(6) TEXT는 아주 긴 문자열을 취급할 때 사용.

(7) DATETIME은 날짜와 시간에 관한 타입.

(8) CONSTRAINT는 제약조건이라는 의미. testTable의 PRIMARY KEY를 id 컬럼으로 지정하겠다는 의미이며, 이 제약조건의 이름을 testTable_PK로 지정한 것이다.

​

그 외의 옵션으로도 많은 값이 올 수 있는데..

중요 기능 두 가지만 더 살펴보자면?

​

CHECK : 값의 범위를 제한한다.

FOREIGN KEY : 다른 테이블과 JOIN할 때 사용한다.

4. ALTER TABLE

테이블의 컬럼 값을 추가하거나 삭제하거나 변경하는 명령을 할 수 있다.

​

(1) ADD

﻿ALTER TABLE 테이블명 ADD 추가할 칼럼명 데이터 유형;﻿

추가된 칼럼은 반드시 마지막에 위치하게 된다.

​

(2) DROP COLUMN

﻿ALTER TABLE 테이블명 DROP COLUMN 삭제할 칼럼명;﻿

(3) ALTER

﻿ALTER TABLE 테이블명 CHANGE 컬럼명 컬럼명 새자료형﻿

칼럼의 데이터 유형, 디폴트 값, NOT NULL 제약조건에 대한 변경을 포함할 수 있음..

타입을 변경할 경우에는 NULL 값이 있는지, 데이터 사이즈가 맞는지 등등을 고려하여 반영해야 한다.

​

(4) ADD CONSTRAINT

﻿ALTER TABLE 테이블명 ADD CONSTRAINT 제약조건명 제약조건 (칼럼명);﻿

컬럼에 제약조건을 추가할 수 있당.
