공간이 자동으로 늘어나고 줄어든다는 점.

import java.util.ArrayList; public class Ex01 { public static void main(String[] args) { ArrayList list = new ArrayList(); list.add("1"); list.add("2"); System.out.println("add: " + list.get(0)); System.out.println("add: " + list.get(1)); list.remove(0); System.out.println("remove: " + list.get(0)); } }

리스트? 배열?

리스트의 특징이자 차이점은

​

배열 : 데이터가 삭제되어도 그대로.

[ 0, 1, 2 ] => 배열 1번째 삭제하면?

[0, " ", 2]

​

리스트 : 데이터가 삭제되면 빈 부분을 땡김.

[ 0, 1, 2 ] => 배열 1번째 삭제하면?

[0, 2]

​

자동으로 땡겨진다는 점!

​

​

+ 정확히는 자바의 배열(array)에서는 원소 삭제가 되지 않는다.

arr[3] = null; 이런 식으로 null 값으로 만들 수는 있지만,

이것은 삭제라기보다는 원소를 비어있는 값으로 업데이트했다는 개념에 더 가깝다.

​

배열에서 원소 삭제를 수행하려면 좀 더 작은 사이즈의 배열을 만들어 해당 원소만 빼고 옮기는 방식으로 처리해야 한다. 이 방법이 번거롭기 때문에 자바에서 List라는 인터페이스를 제공하고 있는 것.

​

List의 대표적인 구현체인 ArrayList의 코드를 직접 열어보면, 얘도 결국 안에서는 배열을 사용하고 있다.

다만 개발자가 직접 배열 원소 삭제를 위해 복사 후 옮기기 등의 노가다를 안 해도 되도록 구현되어 있어 겉보기에는 배열과 다른 것처럼 보이는 것..
