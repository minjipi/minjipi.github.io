오픈스택 수동설치

Glance (이미지 관리 서비스) 설치

​

1) glance DB 생성

[1] controller 노드

데이터베이스 서버에 root 사용자로 연결.

mysql -u root -p

keystone 데이터베이스 생성

CREATE DATABASE glance;

​

keystone 데이터베이스에 적합한 액세스를 부여하고, KEYSTONE_DBPASS 를 적절한 암호로 변경한다.

GRANT ALL PRIVILEGES ON keystone.* TO 'glance'@'localhost' IDENTIFIED BY 'qwer1234';
GRANT ALL PRIVILEGES ON keystone.* TO 'glance'@'%' IDENTIFIED BY 'qwer1234';

​

​

​

2) 사용자 설정 및 엔드포인트 생성

[1] controller

​

source ~/keystone_admin
openstack user create --domain default --password-prompt glance

후 패스워드 설정.

openstack role add --project service --user glance admin
openstack service create --name glance --description "OpenStack Image" image
openstack endpoint create --region RegionOne image public http://컨트롤러 IP:9292 openstack endpoint create --region RegionOne image internal http://컨트롤러 IP:9292 openstack endpoint create --region RegionOne image admin http://컨트롤러 IP:9292

​

​

3) 서비스 설치

[1] controller

dnf --enablerepo=centos-openstack-victoria,powertools -y install openstack-glance

​

4) 서비스 설정

[1] controller

glance-api.conf 파일 백업하고,

mv /etc/glance/glance-api.conf /etc/glance/glance-api.conf.org

​

새로 생성 후

vi /etc/glance/glance-api.conf

아래 내용 추가. (주석해제 및 내용 추가, 변경)

[DEFAULT] bind_host = 0.0.0.0 [glance_store] stores = file,http default_store = file filesystem_store_datadir = /var/lib/glance/images/ [database] connection = mysql+pymysql://glance:설정한 비밀번호@컨트롤러 IP/glance [keystone_authtoken] www_authenticate_uri = http://컨트롤러 IP:5000 auth_url = http://컨트롤러 IP:5000 memcached_servers = 컨트롤러 IP:11211 auth_type = password project_domain_name = default user_domain_name = default project_name = service username = glance password = 설정한 비밀번호 [paste_deploy] flavor = keystone

​

chmod 640 /etc/glance/glance-api.conf chown root:glance /etc/glance/glance-api.conf
su -s /bin/sh -c "glance-manage db_sync" glance
systemctl enable openstack-glance-api.service systemctl restart openstack-glance-api.service
