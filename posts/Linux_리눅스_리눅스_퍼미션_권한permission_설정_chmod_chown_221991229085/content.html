0. 리눅스와 퍼미션(permission)

서버용으로 만들어진 운영체제 리눅스는 여러 사용자가 들어와 사용하는 멀티유저 시스템이다.

​

그런데 이 때문에 한가지 문제가 발생한다!

여러 사용자가 들어오면 서버에 올려진 비밀자료 열람과 그것을 변조,수정,삭제를 할 우려가 높아진다는 것..

​

이 문제의 해결 방안으로 리눅스에 한가지 기능이 추가 되었다.

바로 퍼미션(권한)이라는 기능!

​

퍼미션은 말그대로 시스템에서 유지되는 권한기능인데,

(윈도우도 퍼미션기능이 있긴하지만, 여러사용자들이 들어오는 서버용이 아니기에 리눅스처럼 활성화되어있지 않다.)

이 기능으로 사용자들은 자신한테 읽기퍼미션이 부여된 파일만 읽을 수 있고,

쓰기 퍼미션이 부여된 파일만 쓰기와 수정이 가능하고, 실행퍼미션이 부여된 파일만 실행이 가능하게 된다.

이로써 비밀자료는 퍼미션이 부여된 사용자들에게만 공개할 수 있게 되었다!

​

그리고 이 해결방안에서 그렇담"이 파일은 '누구'꺼지?"라는 소유자 개념이 생겨난다.

앞에서 리눅스는 여러 사용자들이 공동으로 사용하는 운영체제하고 했지만,

만든 사람이 A라도 B가 거의 모든 내용을 수정했다면, B의 것이라고 해도 되는거 아닌가?

하는 분쟁이 일어나는 것이다 !-!

​

2. About 퍼미션

퍼미션에 대한 정보는

ls -al 명령어로 현재위치에 있는 파일들을 상세히 볼 수 있는 아래의 창에서

가장 왼쪽에 위치한 'drwxr - xr -' 으로 확인할 수 있다.

참고로 위 창은

1. 해당 파일에 대한 퍼미션 정보

2. 해당 파일에 link되어 있는 파일이 개수

3. 해당 파일의 User : 파일 생성자

4. 파일의 그룹 : 대부분 파일을 생성한 계정이 속해있는 그룹

​

drwxr-xr-x 1 root root 4096 May 5 09:15 minji.txt

파일Type 퍼미션정보 링크수 소유자 소유그룹 용량 생성날짜 파일이름

- 파일 Type : 파일 종류. -는 일반 파일, d는 디렉터리.

- 퍼미션 정보 : 파일을 읽고 쓰고 실행할 수 있는 접근 권한(퍼미션)을 표시.

rwx: Read, Write, eXecute. 즉 읽거나, 쓰거나, 실행할 수 있다는 것.

'rwx'= 셋 다 가능.

'r-x'= 읽거나 실행할 수는 있지만 쓰기, 즉 수정은 불가능.

​

- 링크 수 : 해당 파일이 링크된 수! 링크는 윈도우의 "바로가기"같은 것이다.

"in [대상파일] [링크파일]" 명령으로 링크파일을

- 소유자 : 해당 파일의 소유자 이름. 리눅스에서 모든 파일은 소유자가 있다. 시스템 간련된 파일은 대부분 root 계쩡이 소유자 이고 일반 파일은 해당 파일을 생성한 사용자가 소유자.

- 소유그룹 : 해당 파일을 소유한 그룹이름. 특별한 변경이 없을 경우 소유자가 속한 그룹이 소유그룹으로 지정된다.

- 용량 : 파일의 용량

- 생성날짜 : 파일이 생성된 날짜

- 파일이름 : 이름.

​

2. 퍼미션 정보

위에서 확인한 퍼미션 정보 rwxr-xr-x 를 자세히 살펴보자.

일단 세자리씩 끊어보면 ( rwx r-x r-x ) 로 읽을 수 있다!

우선 퍼미션 종류에는 3가지가 있다.

또 퍼미션의 사용자 지정에도 3가지.

​

* 퍼미션 종류 , () 괄호에 있는 것이 해당 퍼미션 기호

읽기 ( r ) : 파일의 읽기권한

쓰기 ( w ) : 파일의 쓰기권한

실행 ( x ) : 파일의 실행권한

​

* 퍼미션의 사용자지정

소유자 : 소유자에 대한 퍼미션지정

그룹 : 소유그룹에 대한 퍼미션지정

공개 : 모든사용자들에 대한 퍼미션지정

​

그래서 현재 우리 퍼미션 정보인 rwxr-xr-x 를 해석해 보자면,

이 파일에 대해서 소유자는 읽기(r),쓰기(w),실행(x)을 허용하고,

파일의 소유그룹에 속하고 있는 사용자들은 읽기(r),실행(x)만 허용하고,

이외에 나머지 모든 사용자들도 읽기(r),실행(x)만 허용한다.....

고 이해하면 되겠다.

3. 퍼미션 변경

여러가지 방법이 있지만, 나는 chmod 명령을 사용했다.

chmod [변경될 퍼미션값] [변경할 파일]

여기서 [퍼미션값]을 지정해 줘야하는데, 퍼미션값을 구하는 방법은 다음과 같다.

​

1) 각 퍼미션 기호를 숫자로 변환. ( r = 4 , w = 2 , x = 1 )

r - x 인 경우. 4 0 1

r : 4 (2진수 100)

w : 2 (2진수 10)

x : 1 (2진수 1)

2) 변환한 숫자를 합산.

4 0 1 인 경우 4+0+1 = 5.

​

이런식으로 하나의 퍼미션을 숫자값으로 변환하면 된다.

예시로 rwxr-xr-x 이면 rwx r-x r-x 세자리씩 끊고,

4+2+1 | 4+0+1 | 4+0+1 숫자변환 뒤 합산하면 "755" 라는 퍼미션값이 나온다!.

("755' 로 퍼미션 주세요.")

​

++ 또 다른 예시로 rwx 를 더하면 r(4) + w(2) + x(1) = 7(2진수로 111) 이 되고,

777 이란건 user, group, other의 퍼미션이 777. 즉 rwxrwxrwx.

​

그래서 " chmod 755 minji.txt " 명령을 실행하면 minji.text 파일이 755에 해당되는 퍼미션으로 변경된다.

그리고 디렉토리도 "-R" 옵션을 사용하면 하위 디렉토리의 모든 디렉토리및 파일의 퍼미션을 변경할 수 있다!

​

chmod -R 777 sample.txt ->

-rwxrwxrwx

chmod 755 sample.txt ->

-rwxr-xr-x.

퍼미션 정보가 변경된 것을 확인했다.

​

4. 소유자 변경

파일의 소유자 양도를 해야 될 경우 소유자 변경도 할 수 있다.

chown [변경할 소유자] [변경할 파일]

이 명령으로 소유자뿐만 아니라 소유그룹도 변경할 수 있다.

[변경할 소유자]란에 ".그룹이름" 형식으로 입력하면 끝.

​

minji.txt의 소유자를 "minji"로, 소유그룹을 "minjiland"로 동시에 변경할 경우,

chown minji.minjiland minji.text 라고 입력하면 한 방에 된다!
