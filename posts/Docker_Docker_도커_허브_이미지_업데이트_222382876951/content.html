2. 도커 허브에 레포지터리 생성

📍 도커 허브 로그인 해서 레포지터리 생성.

​

텅 빈 레포지터리가 잘 만들어 졌다면, 이제 이 레포지터리에 이미지를 업로드 해주자.

​

​

1. 그 전에, 업로드 할 이미지 확인

docker image list

​

2. 레포지터리에 올릴 이미지 태그 변경.

docker tag ctomcat:1.0 ghdalswl77/minji-repo:1.0 [도커ID /repo 이름 :버전]

​

3. 이미지 push (이미지를 레포지터리에 업로드)

docker push ghdalswl77/minji-repo:1.0 [도커ID /repo 이름 :버전] ...

📍 denied: requested access to the resource is denied

혹시 위 문구가 뜬다면 도커 로그인을 해주자.

docker login

​

로그인 후 다시 push.

​

결과 : 레포지터리에 이미지 잘 업로드 !

​

​

4. node1과 node2에 이미지를 pull

노드1과 노드2에서도 이 이미지를 내려받는다.

(각각 로그인 후)

docker login #도커 로그인
docker pull ghdalswl77/minji-repo:1.0 #이미지 받기

​

5. 서비스 생성

manager 에서

docker service create --name web -p 8080:8080 ghdalswl77/minji-repo:1.0 [도커ID /repo 이름 :버전]

​

'web' 서비스 생성 확인:

​

scale 주기

docker service scale web=3

​

각 노드들 아이피/경로/1.jsp 로 접속 되는지 확인.
