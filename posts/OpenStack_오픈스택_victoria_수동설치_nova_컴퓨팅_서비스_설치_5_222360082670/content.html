오픈스택 수동설치

nova(컴퓨팅 서비스) 설치

​

1) nova DB 생성

[1] controller 노드

데이터베이스 서버에 root 사용자로 연결.

mysql -u root -p

keystone 데이터베이스 생성

CREATE DATABASE nova_api; CREATE DATABASE nova; CREATE DATABASE nova_cell0;

​

keystone 데이터베이스에 적합한 액세스를 부여.

GRANT ALL PRIVILEGES ON nova_api.* TO 'nova'@'localhost' IDENTIFIED BY 'qwer1234'; GRANT ALL PRIVILEGES ON nova_api.* TO 'nova'@'%' IDENTIFIED BY 'qwer1234'; GRANT ALL PRIVILEGES ON nova.* TO 'nova'@'localhost' IDENTIFIED BY 'qwer1234'; GRANT ALL PRIVILEGES ON nova.* TO 'nova'@'%' IDENTIFIED BY 'qwer1234'; GRANT ALL PRIVILEGES ON nova_cell0.* TO 'nova'@'localhost' IDENTIFIED BY 'qwer1234'; GRANT ALL PRIVILEGES ON nova_cell0.* TO 'nova'@'%' IDENTIFIED BY 'qwer1234';

​

2) 사용자 설정 및 엔드포인트 생성

[1] controller

source ~/keystone_admin
openstack user create --domain default --password-prompt nova

하고 패스워드 설정.

openstack role add --project service --user nova admin
openstack service create --name nova --description "OpenStack Compute" compute
openstack endpoint create --region RegionOne compute public http://컨트롤러 IP:8774/v2.1 openstack endpoint create --region RegionOne compute internal http://컨트롤러 IP:8774/v2.1 openstack endpoint create --region RegionOne compute admin http://컨트롤러 IP:8774/v2.1

​

​

3) 서비스 설치

[1] controller

dnf --enablerepo=centos-openstack-victoria,powertools -y install openstack-nova-api openstack-nova-conductor openstack-nova-novncproxy openstack-nova-scheduler

​

4) 서비스 설정

[1] controller

기존 파일 백업.

mv /etc/nova/nova.conf /etc/nova/nova.conf.org
vi /etc/nova/nova.conf

새로 생성하고, 아래 내용 추가. (주석해제 및 내용 추가, 변경)

​

[DEFAULT] enabled_apis = osapi_compute,metadata transport_url = rabbit://openstack:설정했던 rabbit 비번@controller:5672/ my_ip = 컨트롤러 IP state_path = /var/lib/nova [api_database] connection = mysql+pymysql://nova:설정했던 비번@controller/nova_api [database] connection = mysql+pymysql://nova:설정했던 비번@controller/nova [api] auth_strategy = keystone [keystone_authtoken] www_authenticate_uri = http://controller:5000/ auth_url = http://controller:5000/ memcached_servers = controller:11211 auth_type = password project_domain_name = Default user_domain_name = Default project_name = service username = nova password = 설정할 비번 [vnc] enabled = true server_listen = 0.0.0.0 server_proxyclient_address = 컨트롤러 IP novncproxy_base_url = http://컨트롤러 IP:6080/vnc_auto.html [glance] api_servers = http://controller:9292 [oslo_concurrency] lock_path = /var/lib/nova/tmp [placement] region_name = RegionOne project_domain_name = Default project_name = service auth_type = password user_domain_name = Default auth_url = http://controller:5000/v3 username = placement password = 설정했던 비번
chmod 640 /etc/nova/nova.conf chgrp nova /etc/nova/nova.conf

​

5) DB에 설정 내용 저장

su -s /bin/sh -c "nova-manage api_db sync" nova su -s /bin/sh -c "nova-manage cell_v2 map_cell0" nova su -s /bin/sh -c "nova-manage cell_v2 create_cell --name=cell1 --verbose" nova su -s /bin/sh -c "nova-manage db sync" nova

* su -s 명령어 : 쉘을 변경하여 다른 계정으로 변환.

Shell 의 앞글자 s 옵션이다.

​

예를 들어 su -s /bin/csh - root

이렇게 입력하면 c쉘로 변경해서 root 사용자로 바꿔준다.

su -s /bin/sh -c "nova-manage cell_v2 list_cells" nova

결과:

​

systemctl enable openstack-nova-api.service openstack-nova-scheduler.service openstack-nova-conductor.service openstack-nova-novncproxy.service systemctl restart openstack-nova-api.service openstack-nova-scheduler.service openstack-nova-conductor.service openstack-nova-novncproxy.service
