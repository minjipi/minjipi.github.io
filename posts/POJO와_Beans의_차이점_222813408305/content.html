POJO Classes

- POJO(Plain Old Java Object) 는 자바 객채의 표준(ordinay)이며, Java Language Specification 에 의한 어떤 특정한 엄격한(restriction) 경계가 존재하지 않습니다.

- POJO 는 가독성과 재사용성을 중요시하게 사용됩니다.

- POJO 는 restriction 이 적은 만큼 쓰기도 쉽고 읽기도 쉽기때문에 많이 사용되어 왔습니다.

- POJO 의 3가지 규칙이 존재합니다.

- Exend prespcified classes

- Implement prespecified interfaes

- Contain prespecified annotations

​

​

Beans Classes

- Beans 는 POJO 와 달리 엄격한(restriction) 규칙을 가지고 있습니다.

- 모든 JavaBeans는 POJOs 이지만 POJOs 는 JavaBeans 가 아닙니다.

- Fields 는 접근제어자가 private 이어야만 합니다.

- Fields 는 getter 와 setter 를 가져야 합니다.

- constructor 는 argument 를 가져서는 안됩니다.

- Fileds 는 getter 와 setter 로만 접근되어야만 합니다.

​

​

POJO Classes >

public class Animal { String dog; public String cat; private int legs; public Animal(String dog, String cat, int legs) { this.dog = dog; this.cat = cat; this.legs = legs; } public getDog() { return this.dog; } }

Beans Classes >

public class Animal { private int legs; Animal() { } public setLegs(int legs) { this.legs = legs; } public getLegs() { return legs; } }

Spring Bean 등록

@Bean & @Configuration

개발자가 직접 제어가 불가능한 외부 라이브러리 또는 초기 설정을 위한 클래스를 Bean으로 등록할 때 @Bean을 사용한다.

만약 Bean으로 등록해주어야 하는 클래스가 몇 개 안된다면 하나하나 Bean을 등록하면 되겠지만, 많은 양의 클래스를 일일이 Bean으로 등록하면 상당히 번거로운 작업이 된다.

이에 대한 해결책으로 Spring에서는 Annotiaion을 활용해 Bean을 등록하고 사용하는 방법을 제공한다.

​

Bean 등록을 하려하면 @Bean 어노테이션을 작성하면 된다.

어떤 임의의 클래스를 만들어서 @Bean을 붙인다고 되는 것이 아니라, @Bean을 사용하는 클래스에는 반드시 @Configuration어노테이션을 활용하여 해당 클래스에서 Bean을 등록하고자 함을 명시해주어야 한다.

즉, @Configuration을 작성함으로 클래스가 하나 이상의 @Bean메소드를 제공하고 스프링 컨테이너가 Bean 정의를 생성하고 런타임시 그 Bean들이 요청들을 처리할 것을 선언하게 된다.

​

@Configuration없이 @Bean만 사용해도 Spring Bean으로 등록 된다.

대신 메서드 호출을 통해 객체를 생성할 때 싱글톤을 보장하지 못한다.

그렇기 때문에 Spring 설정 정보를 위해서는 반드시 @Configuration을 사용해주어야 한다.

그리고 빈의 설정을 담당하는 @Configuration 어노테이션도 내부적으로 @Component를 가지고 있어 @Configuration이 붙은 클래스도 Spring의 빈으로 등록이 됨을 알아두어야 한다.

​

위 내용을 요약하자면, @Configuration어노테이션이 붙은 클래스는 스프링의 환경설정을 담당하는 클래스라는 것을 명시하는 것이고, 그 클래스의 내부에서 @Bean이라는 어노테이션이 붙은 메서드는 이 메서드가 생성하는 객체를 Bean객체로 등록하여 스프링이 관리하겠다는 뜻이다. 메서드의 이름은 나중에 Bean객체를 불러올 때 식별자로 사용된다.
