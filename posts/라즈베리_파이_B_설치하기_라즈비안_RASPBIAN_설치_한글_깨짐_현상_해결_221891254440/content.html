3. 근데 라즈베리파이 부팅이 안된다면?

밍.

Raspbian 이미지를 정확히 write 하고, 전원을 연결했는데? 부팅이 안된다의 경우,(내 경우..)

​

라즈베리파이는 SD card 옆쪽으로 LED가 2개, 녹색?초록색과 빨간색이 1개씩 있는데 정상적으로 부팅이 잘 되었을 땐 LED - 녹색 LED가 순차로 점등이 된다.

여기서 빨간색 LED은 POWER 상태를, 녹색 LED는 SD카드 상태를 표시한다. 말인즉슨 빨간색 LED가 안켜지면, 전원이 제대로 되지 않음을 의미하고,

그 반대의 경우 빨간색 LED가 켜졌는데 녹색 LED가 안켜지면 SD 인식이 제대로 안되었음을 의미.

따라서 라즈베리파이의 부팅이 안될때의 원인은 크게 2개다.

​

3-1. 전원 불안

라즈베리파이의 전원은 microUSB 충전기 5V/2.5A 이상을 사용해야 한다. 전류가 2.5A 보다 낮으면 화면이 먹통이거나 처음에 부팅 화면만 나온 뒤 그대로 멈추는 일이 발생! 이때는 전원 스펙을 확인하고 다른 충전기로 교체하면 될 가능성이 높다. 또 부팅이 되었어도 전력이 부족한 경우가 있는데, 화면 우상판에 패널 정보에 노란색 번개 표시가 나타날 때다. 노란색 번개는 undervoltage가 발생했다는 알림인데 요게 나타나면 CPU 로드가 높아지거나 USB에 연결된 디바이스에서 전류를 많이 사용할 경우에 파이가 뻗을 확률이 크다.

​

3-2. SD카드 이미지 문제

이 경우 나는 SD카드를 뺐다가 다시 삽입해 보고, 그래도 안되면 writing 단계에서 문제가 발생했을 수 있다. 특히 부팅 화면에서 노란색 번개 표시가 없이 멈춘다면 이미지가 잘못 씌여졌을 확률이 크니, 이런 경우에는 SD Card에 이미지 writing을 다시 진행해 봐야한다.

이렇게 부팅 성공 화면이 나오면 끝이당ㅎ

이어아래와 같이 몇가지 초기 설정 화면이 뜨는데 그냥 진행하면 된다.

​

1. Expand Filesystem : SD 카드의 전체용량을 사용할 수 있게 함.

2. Change User Password : ‘pi’ 계정의 비밀번호를 변경.

3. Enable Boot to Desktop/Scratch : 부팅시 명령어 환경이 아닌 GUI Windows 환경으로 시작.

4. Internationalisation Options : 언어, 키보드, Timezone 등을 설정.

5. Enable Camera : Raspberry Pi 전용 카메라를 사용할수 있도록 함.

6. Add to Rastrack : rastrack 서버에 사용중인 Raspberry Pi를 등록. 전 세계적으로 서버에 등록된 Raspberry Pi의 현황을 파악할 수 있음.

7. Over Clock : 700MHz 이상의 CPU Clock 으로 변경 가능하게.

8. Advanced Options : Hostname 변경, SSH on/off, SPI on/off, Audio HDMI/3.5mm jack 등을 설정.

​

​

여기서 꼭 해야할 것 [Expand FileSystem]

SD카드를 복사해서 사용하게 되면 사용 중인 용량이 2G가 조금 넘게 되어있다. 초기 상태에서는 2G 밖에 SD를 사용 할 수 없다. 보유한 SD카드가 4G의 SD를 사용하더라도, 8G의 SD를 사용하더라도 2G 정도의 용량 만을 사용 할 수 밖에 없다..SD 전체를 사용하기 위해서는 이 메뉴(Expand FileSystem)를 실행해야 한다.
