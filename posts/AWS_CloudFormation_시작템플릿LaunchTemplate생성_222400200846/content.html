AWS 시작템플릿(LaunchTemplate) 이란?

AWS::EC2::LaunchTemplate

AWS 시작템플릿(LaunchTemplate)이란?

공식사이트에서는 이렇게 말하고 있다.

EC2 인스턴스 시작에 필요한 구성 정보를 모아둔 template,

쉽게말해 EC2 만들때 설정해야 할 많은 설정값들을 미리 모아둔 템플릿이다.

​

이 시작 템플릿에는 버전이 있는데, 코드를 살짝 바꾸고 버전 올려서 배포하는 것과 비슷하게 생각하면 된다.

도커를 해본 사람이면, Dockerfile 과 같은 개념이라고 생각하면 될 것 같다!

​

​

CloudFormation로,

LaunchTemplate 생성

밑져야 본전, 실습에 앞서 구글에 시작쳄플릿의 Type명인

'AWS::EC2::LaunchTemplate'를 검색해 공식 홈페이지의 문서를 참고한다.

​

1. 스택 생성

​

2. 파일(yml) 업로드 준비

​

3. LaunchTemplate.yml 파일 코드:

Description: A sample template Resources: MyEC2Template: Type: AWS::EC2::LaunchTemplate Properties: LaunchTemplateName: MyEC2Template LaunchTemplateData: BlockDeviceMappings: - DeviceName: "/dev/xvda" Ebs: DeleteOnTermination: 'true' VolumeSize: 8 VolumeType: gp2 ImageId: ami-0ba5cd124d7a79612(기본 우분투 이미지 아이디) InstanceType: t2.micro KeyName: 본인 키페어 이름 SecurityGroupIds: - 본인 보안그룹 아이디 UserData: Fn::Base64: !Sub | #!/bin/bash apt-get -y update apt-get install -y apache2 systemctl start apache2

* 아래와 같이 UserData에

시작 템플릿이 생성된 후 실행시키고 싶은 명령들을 작성한다.

​

​

4. 파일 업로드 후,

[시작템플릿] 탭에가서 시작 템플릿이 생성된 것 확인.

​

5. 이 생성된 시작 템플릿으로 인스턴스를 생성해 접속 확인을 해본다.

​

[인스턴스 생성]

기본값으로 [생성]

​

[시작 템플릿 보기] 클릭.

​

아이피를 확인하고, ssh 접속해보자.
