Cluster test

클러스터가 구성이 되었으니

몇 가지 앱을 클러스터에 올리는 간단한 예제를 실행해본다.

​

그 전에, 쿠버네티스는 도커와 다르게 자동완성 기능이 전혀 없어서 아래 명령어로 간단한 명령어 정도는 자동완성이 되게 해준다.

​

💡 쿠버네티스 자동완성 (모두 되지는 않음ㅠㅠ)

source <(kubectl completion bash) echo "source <(kubectl completion bash)" >> ~/.bashrc

​

pod 실행

pod : 가장 작은 쿠버네티스의 단위.

​

도커 이미지를 먼저 받아와도 되지만

실행하면 실행에 필요한 이미지를 알아서 pull 해오니 그냥 실행하도록 했다.

​

[pod 생성]

kubectl run hello-world --image=hello-world -it --restart=Never

hello-world 라는 이름의 간단한 앱을 실행시켜본다.

​

​

결과 :

​

pod 목록 확인 :

kubectl get pods

​

​

pod 삭제:

kubectl delete pod hello-world

​

pod로 톰캣 생성

kubectl run tomcat --image=tomcat -it --restart=Never

​

생성한 pod 로그 확인

kubectl logs tomcat

​

세부 사항 확인

kubectl describe pods nginx

worker2에 노드가 있으니 worker2에서

docker ps -a 해본다.

​

​

다른 노드에는 없다.

​

​

kubectl deployment

pod 보다 큰 개념.

kubectl create deployment --image nginx nginx

​

kubectl get deployment

​

deployment 삭제

kubectl delete deployment nginx

​

​

kubectl deployment replicas

kubectl create deployment --image nginx nginx --replicas=3

​

kubectl get deployment

​

kubectl get pod

​

kubectl describe pod nginx-6799fc88d8-857ff | grep Node

​

​

Scale

kubectl scale --replicas=2 deployment/nginx

​

pod 삭제.

자동으로 scale=4 가 유지가 되는지 확인해 본다.

​

kubectl get pod 로 뽑은 팟 중 2개를 delete 하고 다시 pod 을 확인해보자.

새로운 pod이 2개 생겨 4개를 맞춘다.

​

​

💡 명령어 정리

[ 팟 상태 보기 ]

kubectl get pod -n kube-system kubectl get pods --all-namespaces

​

[ 팟 상태 자세히 보기 ]

kubectl describe pod [팟 이름]

​

[ 노드들 상태 보기 ]

kubectl get nodes

​

[ 노드 상태 자세히 보기 ]

kubectl describe node [노드 이름]
