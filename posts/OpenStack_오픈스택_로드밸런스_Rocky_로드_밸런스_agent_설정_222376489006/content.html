0. 로드밸런싱 ?

로드밸런싱, 로드밸런서는 한마디로 말해서 많은 사용자가 웹사이트를 사용할때 웹사이트의 부하를 분산해주는 부하 분산 처리기라고 할 수 있다. (종류도 여러가지가 있다.)

아래 그림은 오픈스택의 Octavia라는 로드밸런서의 아키텍처이다.

우리는 이 Octavia로 실습하다가 에러로 인해 Rocky 버전 팩스택 설치한 것이고..🥺 쭈글

​

​

정리하자면 :

사용자는 웹사이트를 접속하기 위해 Amphora라는 인스턴스에서 실행되는 https 또는 http 리스너를 통해 웹사이트에 접속을 한다.

이때 접속할 웹사이트들의 집합을 Pool이라고 하고 웹사이트들을 Pool의 Member라고 하고.

그리고, Member들이 잘 살아있는지, 정상적으로 동작을 하는지를 체크하는 모니터링 도구를 Pool Health Monitor라고 한다는 말이다.

​

​

Octavia에서 제공하는 로드밸런서는 아래와 같은 서비스들로 구성되어 있다.

- 로드밸런서

- Amphora 인스턴스

- 리스너(Listener)

- 풀(Pool)

- 멤버(Member)

- 헬스체크 모니터(Health-check Monitor)

​

​

여기서 가장 중요한 역할을 하는건 바로 amphora 라고 불리는 인스턴스이다.

amphora 인스턴스는 Red Hat Enterprise Linux 7.6 기반에 Haproxy 데몬이 설치되어 있다.

그리고, Octavia-agent가 Octavia-server로부터 명령을 받아 Listener를 생성하고 Haproxy에 환경설정을 하고 VIP를 Virtual Network로 생성을 하는 역할을 한다. 그러면 클라이언트는 Octavia-agent에 의해 생성된 VIP를 통해 Haproxy가 연결해주는 VM으로 접근을 하게 되는 것이다. 실습을 하며 다시 생각해보자.

1. 로드밸런싱 수동 설치

팩스택으로 설치했다면 이 섹션은 건너뛴다.

​

​

[컨트롤러 노드]

(1) lbaas 설치

yum --enablerepo=centos-openstack-rocky,epel -y install openstack-neutron-lbaas net-tools

​

(2) lbaas 설정

vi /etc/neutron/neutron.conf 파일에서

service_plugins 뒤에 ,lbaasv2 추가.

예시 👇🏻

service_plugins=qos,trunk,router,metering,lbaasv2﻿

-

​

vi /etc/neutron/neutron_lbaas.conf 파일에서 207번 라인에 다음 내용 추가

service_provider = LOADBALANCERV2:Haproxy:neutron_lbaas.drivers.haproxy.plugin_driver.HaproxyOnHostPluginDriver:default

-

​

vi /etc/neutron/lbaas_agent.ini 파일에서 2번째 라인에 다음 내용 추가

interface_driver = openvswitch

​

(3) 설정 적용

다음 명령어 실행

su -s /bin/bash neutron -c "neutron-db-manage --subproject neutron-lbaas --config-file /etc/neutron/neutron.conf --config-file /etc/neutron/plugin.ini upgrade head"

​

(4) 서비스 재시작

systemctl restart neutron-server

​

​

[컴퓨트 노드]

[네트워크 노드] 둘 다 진행

​

(1) lbaas 설치

yum --enablerepo=centos-openstack-rocky,epel -y install openstack-neutron-lbaas net-tools

​

(2) lbaas 설정

vi /etc/neutron/neutron.conf 파일에서 service_plugins 뒤에 ,lbaasv2 추가

예시 👉🏻 service_plugins=qos,trunk,router,metering,lbaasv2

​

vi /etc/neutron/neutron_lbaas.conf 파일에서 207번 라인에 다음 내용 추가

service_provider = LOADBALANCERV2:Haproxy:neutron_lbaas.drivers.haproxy.plugin_driver.HaproxyOnHostPluginDriver:default

​

vi /etc/neutron/lbaas_agent.ini 파일에서 2번째 라인데 다음 내용 추가

interface_driver = openvswitch

​

(3) 서비스 재시작

systemctl start neutron-lbaasv2-agent systemctl enable neutron-lbaasv2-agent

2. 로드밸런싱 설정

0) 로드밸런스 실습을 위한 인스턴스 준비

:: 인스턴스 2개에 ssh 접속해서 아파치 웹서버 깔아줌.

sudo su - root yum -y install httpd

로드밸런스 실습 시 2개의 인스턴스가 필요한데,

이 때 두 인스턴스를 구별하기 위해 간단한 html 페이지를 작성한다.

​

각각 인스턴스에 접속 후, 아파치 웹서버를 설치하고 /var/www/html 위치에 파일을 만들어준다.

그리고 외부에서 접속이 가능하게 네트워크 방화벽을 끄고, 인스턴스가 사용하는 보안그룹에 80번 포트 허용 규칙도 추가한다. 다 해 주었다면 다음을 진행한다!

​

1) 컨트롤러 노드 관리자 로그인

[ 컨트롤러 노드 ] 관리자로 로그인

source ~/keystonerc_admin

환경변수 적용하여 로그인을 한다.

​

앞으로 종종 다음과 같은 안내가 뜰 경우에 이렇게 로그인을 하면 해결(?)된다.

Auth plugin requires parameters which were not given: auth_url

​

2) lbaasv2 설치 확인

- openstack network agent list 명령어 결과에서 Loadbalancerv2 agent 확인,

- 데시보드 > 시스템 정보 > 컴퓨트, 네트워크 두 개의 컴퓨터에 Loadbalancerv2 agent 설정되어 있는 것 확인.

​

3) 로드밸런서 생성

neutron lbaas-loadbalancer-create --name [로드밸런서 이름] [서브넷] # 예시👇🏻 neutron lbaas-loadbalancer-create --name admin-lb internal-subnet

오픈스택 13 버전의 데시보드에서는 Project > Network > Load Balancers 메뉴를 통해 로드밸런서를 만들수 있지만, 우리는 명령어를 통해 생성해본다.

​

4) 보안 그룹 설정

openstack security group create lbaasv2 openstack security group rule create --protocol icmp --ingress lbaasv2 openstack security group rule create --protocol tcp --dst-port 80:80 lbaasv2

​

5) 로드밸런서에 보안 그룹 적용

openstack port set --security-group [보안그룹 이름] [vip_port_id]

*vip_port_id는 로드밸런서 생성 했을 때 나오는 항목

neutron lbaas-loadbalancer-show [LB 이름]으로 확인 가능.

​

ex) openstack port set --security-group lbaasv2 7697ff69-da98-4793-8d08-7f91b6580751

​

​

6) 로드밸런서에 리스너 설정

neutron lbaas-listener-create --name [리스너 이름] --loadbalancer [로드밸런서 이름] --protocol HTTP --protocol-port 80

아래 예시👇🏻

neutron lbaas-listener-create --name admin-lb-http --loadbalancer admin-lb --protocol HTTP --protocol-port 80

​

​

7) 라운드 로빈 방식의 pool 생성 (실제 서버가 등록될 pool)

neutron lbaas-pool-create --name [pool 이름] --lb-algorithm ROUND_ROBIN --listener [리스너 이름] --protocol HTTP

아래 예시👇🏻

ex) neutron lbaas-pool-create --name admin-lb-http-pool --lb-algorithm ROUND_ROBIN --listener admin-lb-http --protocol HTTP

​

​

8) pool에 서버 등록

neutron lbaas-member-create --name [멤버 이름] --subnet [서브넷 이름] --address [서버의 IP 주소] --protocol-port 80 [pool 이름]

예시👇🏻

ex) neutron lbaas-member-create --name admin-lb-member-01 --subnet internal-subnet --address 10.10.10.4 --protocol-port 80 admin-lb-http-pool
ex) neutron lbaas-member-create --name admin-lb-member-02 --subnet private-subnet --address 10.10.10.10 --protocol-port 80 admin-lb-http-pool

* 대시보드에서 인터널 아이피를 등록. 100.100.100.x 이것 !

​

​

9) pool 확인

neutron lbaas-member-list [pool 이름]

-> 만들어 둔 인스턴스가 있을 것.

​

10) 유동 IP 연결

neutron lbaas-loadbalancer-show [로드밸런서 이름] 에서 vip_port_id 의 값 확인

위 명령어로 vip_port_id 의 값을 확인해서,

openstack floating ip set --port [vip_port_id 값] [유동 IP]

이렇게 유동 IP를 생성한다.

​

예시👇🏻

ex) openstack floating ip set --port 7697ff69-da98-4793-8d08-7f91b6580751 10.10.10.10(아이피)

​

​

3. 로드밸런서 데시보드 설정

1) 설치

yum install -y neutron-lbaas-dashboard

​

2) 설정

vi /etc/openstack-dashboard/local_settings

377번 라인의

'enable_lb': False, 를

'enable_lb': True, 로 변경.

​

systemctl restart httpd

그리고 다시 데시보드를 확인해보면 네트워크, 로드벨런서 페이지가 보일 것 !
