1. ARP 프로토콜

ARP 프로토콜이 하는 일

ARP 프로토콜은 같은 네트워크 대역에서 통신을 하기 위해 필요한

MAC 주소를 IP주소를 이용해서 알아오는 프로토콜.

​

컴퓨터들끼리 통신할 때 IP 주소를 쓰지만,

같은 네트워크 대역에서 실제로 컴퓨터들이 통신할 때 필요한 것은 mac주소다.

​

이더넷 프로토콜로 통신을 할 때, 이더넷 프로토콜에 맥주소가 들어가야 하는데 우리는 IP 주소를 입력함.

그래서 우리가 ip주소를 입력 했을 때 컴퓨터들끼리 서로 상대방의 맥주소를 알아보게 되는데

이 때 'ARP 프로토콜'을 이용하여 상대방의 맥주소를 알아온다.

​

다시말해,

데이터를 보내기 위해서는 7계층부터 캡슐화를 통해 데이터를 보내기 때문에

IP 주소와 MAC 주소가 모두 필요하고,

이 때 IP 주소는 알고 MAC 주소는 모르더라도 'ARP를 통해' 통신이 가능하다는 것.

​

컴퓨터가 계~~속 사용하고 있는 프로토콜!👾

​

​

2. ARP 프로토콜 구조

ARP 프로토콜의 구조

한 줄이 4바이트, 총 28바이트.

Hardware type : 2계층에서 사용할 프로토콜. 왠만하면 여기는 이더넷 프로토콜, 16진수로 = 0001

Protocol type : 여기서쓰는 프로토콜 주소의 타입. = IPv4 = 0800

Hardware Address Length : mac 주소의 길이 : 6바이트 = 06

Protocol Address Length : IPv4 주소 : 4바이트 = 04

Opcode : 어떻게 나타내는지를 나타내는 코드 값인데, 2개 밖에 없음. 내가 요청하고 있는지, 요청에 대한 응답을 나타내고 있는지. 물어보는지 응답하는지! 1로 작성해서 보내면, 2로 셋팅해서 응답을 해준다. 0001 혹은 0002.

---

보통 여기까지는 고정 값이고...

---

Source Hardware Address : 출발지 맥주소를 쓰는 6바이트 자리.

Source Protocol Address : IPv4 주소. 2바이트 씩 2개, 4바이트.

Destination Hardware Address : 목적지 맥주소.

Destination Protocol Address : 목적지 IP 주소.

​

자 그럼,

이 ARP 프로토콜을 이용해서 어떻게 상대방의 맥주소를 알아내는지 자세히 살펴보자.

3. ARP 프로토콜의 통신 과정

IP주소로 MAC주소를 알아오는 과정

아래 그림은,

A 컴퓨터가 C컴퓨터와 통신을 하려고 C의 맥주소를 알아와야 하는데, IP만 알고 있는 상황이다.

​

1. 먼저 A가 상대방에게 ARP 요청 프로토콜을 만들어서 인캡슐레이션을 해서 보낸다.

3계층. 2계층 프로토콜인 이더넷 프로토콜을 붙여 보낸다.

여기서 목적지 MAC주소를 써야하는데!

이더넷 프로토콜 구조가 어떻게 되더라?...

목적지 MAC주소, 출발지 MAC주소가 필요한데. 모르는데..!

​

그래서 일단, ARP 요청 프로토콜을 만들 때 모르는 목적지 맥주소는 00 00 00 00 으로 비워둔다.

​

IP 주소는 알고 있으니까 적어서 보낸다.

'이 IP 주소인 친구는 나한테 맥주소 좀 알려줘' 라고 작성해서 보냄!

이걸 이더넷 프로토콜 인캡슐레이션해서 보낸다.

마찬가지로 목적지 MAC 주소를 모르기 때문에

FF FF FF FF FF FF 로 작성한다.

이게 이진수로 봤을 땐 1로 꽉 채운 상태인데..

​

IP 주소에서도 뒷 부분을 전부다 1로 꽉 채우면 브로드캐스트 주소였는데,

여기서도 1로 꽉 채우면 브로드캐스트 주소가 되어서

이 프로토콜을 같은 네트워크 대역에 있는 모두에게 보낸다.

왜냐면? C가 어딨는지 모르니까.
