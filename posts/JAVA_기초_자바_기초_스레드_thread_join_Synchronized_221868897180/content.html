0. Thread 란?

어떠한 프로그램내에서, 특히 프로세스 내에서 실행되는 흐름의 단위.

쓰레드를 알기 위해서는, 우선 프로세스Process에 대해 알아야 한다.

Java, C/C++ 등 프로그래밍 언어로 소스를 짜서 만든 것을 프로그램이라 하고,

이 프로그램을 실행 시켜서 동작하게 만들면, 그것을 프로세스라 한다.

​

이 프로세스는 보통 하나의 루틴(프로그램 처리 경로)을 가지고 있다.

이 루틴은 직렬적이다. 어떠한 일을 수행하는 것에 있어 프로그래머가 원하는 순서대로 일을 처리한다는 건데, 생각해보면 굳이 앞뒤 순서가 필요 없는 일들이 있을 때 분리해서 동시에 처리하고 싶은 생각이 들 때가 있다! 이 때 자바에서 사용할 수 있는 것이 바로 '스레드'다.

자바 쓰레드를 이용하면 하나의 프로세스에서도 병렬적으로 처리, 즉 여러 개의 처리 루틴을 가질 수 있다.

단순 반복의 코드를 실행할 때도 여러 개의 쓰레드를 만들어서 분리 시킨 뒤, 결과 데이터를 받아 합치면 그만큼 시간을 절약할 수 있다.

​

또 다른 스레드의 특성으로는,

- 메모리를 할당받아 실행 중인 프로그램.

- 프로세스 내의 명령어 블록으로 시작점과 종료점을 가짐.

- 실행중에 멈출 수 있으며, 동시에 수행 가능.

등이 있다!

1. Thread 생성

Thread join, Synchronized.

- 2가지 방법 :

a. Thread 클래스 이용. 직접 상속 받아 스레드 생성.

b. Runnable 인터페이스를 구현해서 생성.

스레드의 구현은 Thread 클래스를 상속받는 방법과 Runnable 인터페이스를 구현하는 방법 두 가지가 있다. 기본적으로 Thread 클래스는 Runnable 인터페이스를 구현한 것이기 때문에 두 방식의 차이가 크진 않다.

Runnable 인터페이스를 구현하면 원하는 기능을 추가할 수 있는데, 해당 클래스를 수행할 때 별도의 스레드 객체를 생성해야 한다. 또 자바는 다중 상속을 인정하지 않아서 스레드를 사용할 때 이미 상속받은 클래스가 있다면 Runnable 인터페이스를 구현해야한다.

a-3)

동기화(Synchronized) 작업이 필요한 상황

  • 하나의 객체를 여러 스레드에서 동시에 사용할 경우

  • static으로 선언한 객체를 여러 스레드에서 동시에 사용할 경우

새로운 정적 메소드인 setIndex()를 코드에 추가했다.

이 키워드가 있는 한 setIndex()를 쓰레드들이 동시에 접근할 수 없고, 한번에 하나씩만 접근할 수 있다.

public class ThreadTest extends Thread { private static int index = 0; private int id = -1; public ThreadTest(int id) { this.id = id; } public void run() { System.out.println(id + "번 스레드 동작 중"); Random r = new Random(System.currentTimeMillis()); try { long s = r.nextInt(1000); ThreadTest.sleep(s); setIndex(); } catch (InterruptedException e) { e.printStackTrace(); } System.out.println(id + "번 쓰레드 동작 종료"); } public synchronized static void setIndex() { index++; } public static void main(String[] args) { System.out.println("==== 메인 함수 시작 ===="); ArrayList<Thread> threadList = new ArrayList<Thread>(); for (int i = 0; i < 10; i++) { ThreadTest test = new ThreadTest(i); test.start(); threadList.add(test); } for (int i = 0; i < threadList.size(); i++) { try { threadList.get(i).join(); } catch (InterruptedException e) { e.printStackTrace(); } } System.out.println("current Index: " + index); System.out.println("==== 메인 함수 종료 ===="); } }

b-3. 람다식 활용

Runnable 인터페이스는 run()이라는 하나의 추상 메소드만을 가지고 있다.

따라서 Runnable은 함수형 인터페이스다. 함수형 인터페이스는 람다식으로 더욱 간결하게 표현할 수 있음.만약 실행할 문장이 한문장 뿐이라면, 단 한 줄의 코드로 쓰레드의 생성부터 실행까지 모두 가능하다.

public class RunnableTest { public static void main(String[] args) { new Thread(() -> { ** 수행할 문장들 }).start(); Thread thread = new Thread(() -> System.out.println("쓰레드가 실행됩니다.")); thread.start(); thread.setName("쓰레드"); System.out.println(thread.getName()); } }

Thread 의 상태 6가지

2.5

a. NEW : 스레드가 생성되었지만 스레드가 아직 실행할 준비가 되지 않았음

b. RUNNABLE : 스레드가 실행되고 있거나 실행준비되어 스케쥴링은 기달리는 상태

c. WAITING : 다른 스레드가 notify(), notifyAll()을 불러주기 기다리고 있는 상태(동기화)

d. TIMED_WAITING : 스레드가 sleep(n) 호출로 인해 n 밀리초동안 잠을 자고 있는 상태

e. BLOCK : 스레드가 I/O 작업을 요청하면 자동으로 스레드를 BLOCK 상태로 만든다.

f. TERMINATED : 스레드가 종료한 상태.

​

스레드 상태는 JVM에 의해 기록 관리된다.

3. Daemon Thread

- 동일한 프로세스 안에서 다른 스레드의 수행을 돕는 스레드로 다른 스레드를 서비스 해주면서 다른 스레드가 모두 종료되면 자신도 종료되는 스레드

- 프로그램이 종료되는 것을 막지 않으며 가비지 컬렉터나 메인 스레드가 데몬 스레드.

- 스레드를 생성하고 setDaemon(true)를 설정하면 됨.

- 스레드가 시작하기 전에 설정해야함.

4. 우선 순위(Priority)

: 2개 이상의 스레드가 동작 중일 때 우선 순위를 부여하여,

우선 순위가 높은 스레드에게 실행의 우선권을 부여할 수 있다.

2개 이상의 스레드가 동작 중일 때 우선 순위를 부여하여,

우선 순위가 높은 스레드에게 실행의 우선권을 부여할 수 있다.

​

: 우선 순위를 지정하기 위한 상수를 제공.

- static final int MAX_PRIORITY : 우선순위 10 - 가장 높은 우선 순위

- static final int MIN_PRIORITY : 우선순위 1 - 가장 낮은 우선 순위

- static final int NORM_PRIORITY : 우선순위 5 - 보통의 우선 순위

​

: 스레드 우선 순위는 변경 가능.

- void setPriority(int priority)

- int getPriority()

​

: main()스레드의 우선 순위 값은 초기값이 5.

​

: JVM의 스케쥴링 규칙

- 철저한 우선 순위 기반

- 가장 높은 우선 순위의 스레드가 우선적으로 스케쥴링

- 동일한 우선 순위의 스레드는 돌아가면서 스케쥴링(라운드 로빈)

6. 멀티 스레드(Multi Thread)

: 여러 개의 스레드가 동시에 수행되면서 공유할 수 있을때 공유되는 부분은 상호 배타적으로 사용되어야 한다.

​

- Dead Lock 이슈

: 멀티 스레드를 사용할 때 주의할 점중의 하나로, 프로그램에서 스레드를 잘못 만들면 프로그램의 수행이 이루어 지지 않고 무한 수행하는 Dead Lock을 만들 수 있음.

​

- 임계 영역(critical section)

- 공유 자원을 사용하는 코드영역을 임계영역이라 합니다.

- 이 부분에서는 공유 자원을 동시에 수정할 수 없도록 상호 배타적으로 실행될 수 있도록 작성되어야함.

​

- 자바에서 상호배제 문제를 해결하는 방법

: 자바는 한 순간에 하나의 스레드만 실행할 수 있는 synchronized method 제공.

한 스레드가 synchronized method를 수행 중이면 다른 스레드는 대기함.

​

- 처리방법

- 공유 자원에 접근하는 메소드의 앞에 synchronized 메소드로 지정.

- 공유 자원을 사용하는 영역을 synchronized(객체명)의 블록으로 지정.
