1. 움직이는 캐릭터가 창 (window frame) 을 넘어가지 않는 것.

​

기존 코드에서는 캐릭터가 왼쪽으로 움직일 때,

window frame 을 넘어 무한정 왼쪽 방향으로 갔던 문제를 고치려고 한다.

화질 왜 이러는지🥺

​

현재 프레임 사이즈가 가로 -200px 에서 +200px 이니까

캐릭터가 대강 20px라고 가정해서-

캐릭터의 x축 위치가 -180에 다다랐을 때 캐릭터가 더 이상 앞으로 가지 못하도록 만들어 보자.

​

[ 기존코드 ] public int playAnim(JLabel lbl, int imageIndex) { if (ImageService.walkingLeftImageName.length <= imageIndex) { imageIndex = 0; } lbl.setIcon(new ImageIcon("./src/img/" + ImageService.walkingLeftImageName[imageIndex])); lbl.setLocation(lbl.getLocation().x + MOVING_DIRECTION * speed, lbl.getLocation().y); return ++imageIndex; } }

우선 기존 코드를 설명하자면,

lbl.setLocation(lbl.getLocation().x + MOVING_DIRECTION * speed, lbl.getLocation().y);

return ++imageIndex;

x위치를 받아와서, 움직이는 방향*속도를 더한 값이 x위치값, y값.

그리고 프레임으로 쪼개진 이미지가 하나 씩 돌아갈 수 있게 ++imageIndex 가 되어 있다.

​

해결책을 생각해 보면...

1. 로케이션의 X 값이 -180 이 넘어가면(보다 작으면) : 값을-180 으로 넣고

2. -180을 넘어가지 않으면 그냥 기존 코드대로(평상시 모드) 값을 넣어주기.

​

여기서 한 단계 더 자세하게 생각해보면 :

1. if (lbl.getLocation().x 가 -180 보다 작아지면?)

{ lbl.getLocation().x를 -180으로 고정한다. }

2. else (lbl.getLocation().x 가 -180보다 크면?)

{ lbl.setLocation(lbl.getLocation().x + MOVING_DIRECTION * speed, lbl.getLocation().y); }

​

그 다음.. 이거 이렇게 하면 될까요? 물어보고,

아래와 같이 코드로 구현한다!

[ 변경 코드 ] public int playAnim(JLabel lbl, int imageIndex) { if (ImageService.walkingLeftImageName.length <= imageIndex) { imageIndex = 0; } lbl.setIcon(new ImageIcon("./src/img/" + ImageService.walkingLeftImageName[imageIndex])); if (lbl.getLocation().x < -180) { lbl.setLocation(lbl.getLocation().x, lbl.getLocation().y); } else { lbl.setLocation(lbl.getLocation().x + MOVING_DIRECTION * speed, lbl.getLocation().y); } return ++imageIndex; } }

이렇게 하면 영상처럼 캐릭터가 예상했던 위치에서 더 창을 뚫고 나가지 않고 멈춘다!

#setLocation

아래는 자바에서 자주 쓰이는(?) 메소드들 중 몇 가지다.

나는 이번 공부를 하기 전까지 setLocation과 getLocation의 차이를 명확하게 알지 못했었다.

Point getLocation()

컴퍼넌트의 위치를 리턴

Point getLocationOnScreen()

화면의 좌표 공간 위치 리턴

String getName()

컴퍼넌트의 이름을 리턴

void setsize(int width, int height)

사이즈를 'width' 및 'height'로변경

void setVisible(boolean b)

'b'에 의해 컴퍼넌트 표시 or 비표시 설정

void setLocation(int x, int y)

이 컴퍼넌트를 새로운 위치로 이동

Point getLocation()

컴퍼넌트의 위치를 리턴

setBounds(int x, int y, int width, int height)

위치, 너비, 높이 지정

add(컴포넌트)

컴포넌트를 부착

getLocation : get 이라는 의미 그대로 값을 받아와 쓰는 것.

setLocation : 내 식대로 표현하자면 세팅한다 할 때 의미 그대로 뭔가 세팅 하는 것.

if (lbl.getLocation().x < -180) { lbl.setLocation(lbl.getLocation().x, lbl.getLocation().y); }

그래서 위 코드 중 이 부분을 해석하자면-

if (받아온.x 값 < -180) 일 때 { lbl.값을 지정해준다(받아온.x값, 받아온.y값); }

import java.awt.BorderLayout; import java.awt.Dimension; import javax.swing.*; import java.awt.FlowLayout; import java.awt.event.ActionEvent; import java.awt.event.ActionListener; import javax.swing.ImageIcon; import javax.swing.JButton; import javax.swing.JFrame; import javax.swing.JLabel; import javax.swing.JPanel; import javax.swing.event.ChangeEvent; import javax.swing.event.ChangeListener; import java.awt.BorderLayout; import java.awt.Dimension; import java.awt.event.ComponentAdapter; import java.awt.event.ComponentEvent; import javax.swing.JFrame; import javax.swing.JPanel; import javax.swing.JTabbedPane; public class Client { public static void main(String[] args) { JPanel panel = new JPanel() { @Override public Dimension getPreferredSize() { return new Dimension(200, 200); } }; panel.addComponentListener(new ComponentAdapter() { @Override public void componentResized(ComponentEvent e) { System.out.println("Resized to " + e.getComponent().getSize()); } @Override public void componentMoved(ComponentEvent e) { System.out.println("Moved to " + e.getComponent().getLocation()); } }); JTabbedPane tabbedPane = new JTabbedPane(); tabbedPane.addTab("test", panel); JFrame frame = new JFrame(); frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); frame.setLayout(new BorderLayout()); frame.add(tabbedPane); frame.pack(); frame.setVisible(true); } }
