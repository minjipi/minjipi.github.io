​

🧞‍♂️CloudFormation 이란?

CloudFormation은 AWS 리소스를 자동으로 생성해 주는 서비스이다.

사용하려는 AWS 리소스를 템플릿 파일로 작성하면, CloudFormation이 이를 분석해서 AWS 리소스를 생성한다.

이렇게 생성된 리소스를 스택이라고 한다.

​

​

🕹 CloudFormation,

어떻게 작동할까?

CloudFormation은 아래 그림과 같이 작동한다.

템플릿 작성, 템플릿 업로드, 스택 생성, 스택 설정 및 리소스 생성의 4단계이다.

자세한 작동방식은 AWS 공식사이트 를 참고하자.

​

​

❇️CloudFormation, 어떻게 만들지?🧐

템플릿은 JSON이나 YAML 형식으로 만들 수 있다.

해당 포스팅에서는 YAML 형식으로 진행할 예정이다!

템플릿의 형식은 CloudFormation 디자이너에서 JSON에서 YAML로 또는 그 반대로 자유롭게 변경할 수 있다.

템플릿에 선언된 리소스를 그래픽으로 표시하고 편집할 수 있는 기능도 제공한다.

​

# 파라미터 생성

이 파일에 적힌대로만 딱! 만들어지게 하는것이 아니라,

만들 때 보기를 제공하는 기능이다.

형식은 다음과 같다.

~~Type:!Ref ~~Parameter Parameters: ~~Parameter:

형식을 참고해 한번 생성 해보고, 결과를 확인해보자.

AWSTemplateFormatVersion: "2010-09-09" Description: A sample template Resources: MyEC2Instance: Type: "AWS::EC2::Instance" Properties: ImageId: "ami-00f90da52b0b4dd7e" #내 이미지 아이디 InstanceType: !Ref InstanceTypeParameter KeyName: !Ref KeyNameParameter BlockDeviceMappings: - DeviceName: /dev/sdm Ebs: VolumeType: gp2 DeleteOnTermination: false VolumeSize: 8 Parameters: InstanceTypeParameter: Type: String Default: t2.micro AllowedValues: - t2.micro - m1.small - m1.large KeyNameParameter: Type: String Default: key #내 키페어 중, 이 인스턴스에 기본 설정할 키페어 AllowedValues: - key #내 키페어 이름1 - test #내 키페어 이름2 VolumeSizeParameter: Type: Number Default: 8

​

이렇게 키는 뭘 사용할 것인지, 볼륨 크기는 어떻게 할 것인지 선택할 수 있는 파라미터가 만들어진다.

[ TargetGroup, LoadBalancer, Listener 전체 생성 ]

AWSTemplateFormatVersion: "2010-09-09" Description: A sample template Resources: MyTargetGroup: Type: AWS::ElasticLoadBalancingV2::TargetGroup Properties: HealthCheckEnabled: True HealthCheckIntervalSeconds: 30 HealthCheckPath: / HealthCheckPort: 80 HealthCheckProtocol: HTTP HealthCheckTimeoutSeconds: 5 HealthyThresholdCount: 5 Matcher: HttpCode: 200 Name: myTargetG Port: 80 Protocol: HTTP TargetType: instance UnhealthyThresholdCount: 2 VpcId: vpc아이디 StackELB: Type: AWS::ElasticLoadBalancingV2::LoadBalancer Properties: IpAddressType: ipv4 Name: Stack-elb SecurityGroups: - 보안그룹 아이디 Subnets: - 서브넷 아이디 - 서브넷 아이디(두개가 달라야함) Type: application HTTPlistener: Type: AWS::ElasticLoadBalancingV2::Listener DependsOn: - MyTargetGroup - StackELB Properties: DefaultActions: - Type: "forward" ForwardConfig: TargetGroups: - TargetGroupArn: !Ref MyTargetGroup LoadBalancerArn: !Ref StackELB Port: 80 Protocol: "HTTP"

​

MyTargetGroup은 인스턴스가 먼저 생성된 다음 작성되어야 하는 부분이 있다.

그래서 (위에 초록색 박스에 표시해둔 부분)

MyTargetGroup에 DependsOn: MyEC2Instance 를 추가했고,

Targets: - Id: !Ref MyEC2Instance 추가해줬다.
