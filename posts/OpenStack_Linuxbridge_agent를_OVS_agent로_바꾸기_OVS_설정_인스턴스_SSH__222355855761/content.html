OVS 설정

​

[ controller ]

ovs-vsctl add-br br-int
ovs-vsctl add-br br-[네트워크 장치 이름] ovs-vsctl add-br br-enp2s0

* ifcfg 는 빼고...

​

ovs-vsctl add-port br-[네트워크 장치 이름] [네트워크 장치 이름]﻿

​

vi ifcfg-ens33 DEVICE=ens33 TYPE=OVSPort DEVICETYPE=ovs OVS_BRIDGE=br-[네트워크 장치 이름] ONBOOT=yes vi ifcfg-br-[네트워크 장치 이름] DEVICE=br-[네트워크 장치 이름] DEVICETYPE=ovs TYPE=OVSBridge BOOTPROTO=static IPADDR=[컨트롤러 IP] NETMASK=255.255.255.0 GATEWAY=[게이트웨이] DNS1=8.8.8.8 ONBOOT=yes

​

systemctl restart network

​

네트워크 재시작하면 원격 연결 접속 끊어지고 다시 접속 불가.

가상머신 콘솔에서 ifconfig 확인 해보면..

​

ovs-vsctl show

로 확인해보니 설정이 다 사라져 있음.

​

아래 두 개 다시하고

ovs-vsctl add-br br-[네트워크 장치 이름] ovs-vsctl add-port br-[네트워크 장치 이름] [네트워크 장치 이름]

​

재시작.

systemctl restart network

​

최종 확인 ::

ens33한테는 IP가 없고, br-ens33에 IP가 설정되어 있음.

ssh나 putty 연결 다시 가능해짐.

​

vi /etc/neutron/plugins/ml2/openvswitch_agent.ini
[ovs] bridge_mappings = provider:br-ens33 local_ip =[컨트롤러IP]
systemctl restart neutron-openvswitch-agent

​

vi /etc/neutron/plugins/ml2/ml2_conf.ini

들어가서 아래 부분 수정.

tenant_network_types = vxlan
systemctl restart neutron-server

​

vi /etc/neutron/plugins/ml2/openvswitch_agent.ini

들어가서 맨 마지막에 다음 추가.

[agent] tunnel_types = vxlan prevent_arp_spoofing = True
systemctl restart neutron-openvswitch-agent

네트워크 다시 생성.

​

​

노트북에서 키 파일과 할당받은 아이피로 ssh 접속해보기.

키 파일 저장해둔 곳에서 아래 명령어 입력.

ssh centos@192.168.1.126 -i linux.pem [IP] [key file]

​

** ssh 접속 시,

Permissions 0644 for 'linux (1).pem' are too open. 뜬다면 chmod 600 주기.

chmod 600 linux.pem
