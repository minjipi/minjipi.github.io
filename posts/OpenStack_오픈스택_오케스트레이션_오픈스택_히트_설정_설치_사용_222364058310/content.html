(2) 사용자 및 엔드포인트 추가

[ controller ]

openstack user create --domain default --password-prompt heat 패스워드 설정 후
openstack role add --project service --user heat admin
openstack role create heat_stack_owner
openstack role create heat_stack_user

예시 화면

openstack role add --project admin --user admin heat_stack_owner
openstack service create --name heat --description "Openstack Orchestration" orchestration
openstack service create --name heat-cfn --description "Openstack Orchestration" cloudformation

​

heat_api=[네트워크 노드 IP]

​

openstack endpoint create --region RegionOne orchestration public http://$heat_api:8004/v1/%\(tenant_id\)s
openstack endpoint create --region RegionOne orchestration internal http://$heat_api:8004/v1/%\(tenant_id\)s
openstack endpoint create --region RegionOne orchestration admin http://$heat_api:8004/v1/%\(tenant_id\)s
openstack endpoint create --region RegionOne cloudformation public http://$heat_api:8000/v1
openstack endpoint create --region RegionOne cloudformation internal http://$heat_api:8000/v1
openstack endpoint create --region RegionOne cloudformation admin http://$heat_api:8000/v1

​

openstack domain create --description "Stack projects and users" heat
openstack user create --domain heat --password servicepassword heat_domain_admin
openstack role add --domain heat --user heat_domain_admin admin

2) 오픈스택 히트 사용해보기

(1) yml,yaml 파일

: xml, json처럼 데이터를 전달하기 위한 일종의 양식, 사람이 쉽게 읽을 수 있는 데이터 직렬화 양식.

​

(2) 샘플 파일 실행해보기

vi sample.yml
heat_template_version: 2018-08-31 description: Heat Sample Template parameters: ImageID: type: string description: Image used to boot a server NetID: type: string description: Network ID for the server resources: server1: type: OS::Nova::Server properties: name: "Heat_Deployed_Server" image: { get_param: ImageID } flavor: "m1.small" networks: - network: { get_param: NetID } outputs: server1_private_ip: description: IP address of the server in the private network value: { get_attr: [ server1, first_address ] }

이렇게 작성하고 저장 후 나가서

​

아래 두 명령어를 사용해서 ID, 이름 확인 후

​

해당 ID 를 사용하여 아래 명령어로 stack 생성.

openstack stack create -t sample.yml --parameter "ImageID=[이미지 ID];NetID=[internal 네트워크의 ID]" Sample-Stack

​

생성된 것 확인은 다음 명령어로 :

openstack stack list

​

그리고 조금 뒤 대시보드를 확인하면

이렇게 인스턴스가 생성되어있다.

​

​

​

이번 포스팅인 Heat 를 이용해서 로드밸런스를 생성하고 자동으로 인스턴스가 생성되며, 자동으로 생성된 인스턴스가

​

로드밸런스 멤버풀에 추가되며, 그렇게 추가되어 동작하는 인스턴스에 cpu 사용률을 측정해서 높아지면 자동으로 추가 인스턴스가 생성되는 것.

<오픈스택 록키버전 설정, 설치> 는 원래 없어야할 부분이다.

수동으로 하나하나 설치해서 모든 서비스를 이용해 보려고 했는데

(히트, 로드밸러스, 히트로 로드밸런스도 자동으로 생성)

로드밸런스가 안되서 문제가 생긴 것.

그래서 굳이 다시 록키버전으로 재 설치를 하면서 록키버전 내용을 정리했다.
