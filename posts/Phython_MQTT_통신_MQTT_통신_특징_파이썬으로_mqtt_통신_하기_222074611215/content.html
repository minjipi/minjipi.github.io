MQTT

MQTT 란?

MQTT란 M2M, IOT를 위한 프로토콜로서,

최소한의 전력과 패킷량으로 통신하는 프로토콜이다.

출처 : https://www.eclipse.org/community/eclipse_newsletter/2014/october/article2.php

​

IOT와 모바일 어플리케이션 등의 통신에 매우 적합한 프로토콜. 독특한 점이라면

MQTT는 HTTP, TCP등의 통신과 같이 클라이언트-서버 구조로 이루어지는 것이 아닌

Broker, Publisher, Subscriber 구조로 이루어져있다는 것! 여기에 대해 조금 더 알아보자.

​

MQTT의 특징

MQTT의 특징인 통신 방식에 대해 알아보자.

PUBLISHER <--> BROKER <--> SUBSCRIBER

기존 방식과 비교하자면 'BROKER' 가 '서버' 역할을 하는 것.

BROKER 를 거쳐 다양한 장비들이 통신을 주고 받는 형태다.

SUBSCRIBER 가 중간에서 서버 역할을 하는 것이고,

메시지를 보내고 싶어하는 쪽이 PUBLISHER.

아래는 MQTT 를 이해하기 위한 몇가지 키워드들인데 일단 읽어본 다음 예제를 통해 더 이해하도록 하자.

​

# Client

MQTT 의 broker 에 연결된 모든 것.

​

# BROKER

서버와 클라이언트 사이의 중간 역할.

모든 메세지를 수신, 필터링, 메시지 구독하는 사람 결정 클라이언트에게 메세지를 보내는 역할.

​

# Publish

topic을 지정하여 topic 을 subscribe 하고 있는 클라이언트에게 메세지를 보낸다.

​

# Subscribe

topic을 구독하여 topic으로 publish된 메시지를 받는다.

​

# Topic

MQTT의 topic은 슬래쉬(/)를 분리 문자로 사용하여 폴더 및 파일과 유사한 계층 구조를 가진다.

camera/data1 : ex)카메라로부터 받는 데이터 sensor/data1﻿house : ex)습도 센서로부터 받는 데이터 house/room house/room/main-light house/room/left-light

​

MQTT 동작 구조

트위터 서비스의 동작구조를 떠올려보자.

트위터에서 사용자는 다른 사용자를 follow 할 수 있다. 그럼 다른 사용자가 생성하는 메시지(트윗)를 받아볼 수 있다. 그리고 스스로 메시지를 생성할 수도 있고.. 그럼 자신을 follow하는 사용자에게 메시지가 전달된다. MQTT도 이와 유사한 동작구조를 갖는다.

MQTT 시스템에 참여하는 MQTT 클라이언트는 메시지 발행(publish, 트윗에 해당), 메시지 구독(subscribe, follow에 해당) 두 가지 동작을 할 수 있다.

MQTT 클라이언트가 메시지를 특정 채널(Topic, 토픽)에 발행하면,

이 채널을 구독한 모든 클라이언트에게 메시지가 전달되는 것이다.

중간에서 메시지를 수집, 재분해 하는 작업은 MQTT 브로커 담당이다.

1) subscriber 코드

import paho.mqtt.client as mqtt # subscriber callback def on_message(client, userdata, message): print("message received ", str(message.payload.decode("utf-8"))) print("message topic=", message.topic) print("message qos=", message.qos) print("message retain flag=", message.retain) broker_address = "192.168.100.88" client1 = mqtt.Client("client1") //구독자 이름 client1.connect(broker_address) //broker 주소 등록 client1.subscribe("camera/data") //등록하고픈 토픽 지정 client1.on_message = on_message client1.loop_forever()

on_message ​함수를 만들고 콜백함수로 등록시킨다.
