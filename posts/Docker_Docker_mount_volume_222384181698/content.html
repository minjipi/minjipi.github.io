[Docker]

도커 mount volume

​

​

​

일단 볼륨을 쓰는 이유는

컨테이너에서 생성되는 데이터가 컨테이너가 사라져도 남아있게하기 위해서.

​

방법 1)

volume타입 -> 도커 볼륨 이용

​

방법 2)

bind타입 -> 호스트 PC의 저장소를 공유하는 방식.

​

여기까지는 도커 볼륨과 다를게 없어 보이지만,

스웜 볼륨에서는 '여러 컴퓨터에 컨테이너가 따로 생긴다는 점'에 주목하자.

​

​

방법 1) 첫번째, 도커 볼륨을 이용하는 방식을 보면

docker volume ls 로 볼륨을 확인하고(3가지 노드 전부 볼륨을 다 지우고)

아래 명령어로 서비스를 만들고, 스케일을 늘려서, 각 노드에 컨테이너가 생성되게 하면

docker service create --name web \ --mount type=volume,source=my_vol,target=/usr/local/tomcat/webapps/minji \ -p 8080:8080 tomcat:latest

docker volume ls 했을 때, my_vol 이라는게 각 노드에 생성된다.

그래서 컨테이너가 사라져도 my_vol이라는 볼륨이 남아있기 때문에, 컨테이너에서 생성되는 파일들은 영구적으로 남아있는 것.

​

그런데 각 컴퓨터에 각자 my_vol 이라는 도커 볼륨이 생기는 것이기 때문에 각 노드의 컨테이너들이 파일을 공유하는게 아니라 각자 각각의 my_vol 이라는 볼륨에 파일이 저장되는 것이다.

​

그래서 각자 따로 관리해야 되는 파일이 있을 때는 도커 볼륨으로 서비스를 만들어서 쓰는 것이 좋다.

​

​

​

​

방법 2) 두번째 방식, 호스트 PC의 폴더를 공유해서 쓰는방식.

docker service create --name web \ --mount type=bind,source=/vol,target=/usr/local/tomcat/webapps/minji \ -p 8080:8080 tomcat:latest

위 명령어를 이용해 호스트PC의 폴더를 공유해서 쓰면

각 노드에 /vol이라는 폴더가 각각 있어야 하고 그냥 사용하면 위에 도커 볼륨과 마찬가지로 각각 서로 다른 데이터를 저장하겠지만,

nfs와 같은 폴더 공유를 이용해서 하나의 노드가 폴더를 공유하고 다른 노드들은 nfs로 마운트해서 쓰면 하나의 폴더를 각 컨테이너들이 공유해서 쓰기 때문에 파일을 한번만 변경해도 각 노드의 모든 컨테이너들에 적용이 될 수 있다는 것. jsp 파일 같은 경우는 bind 타입의 볼륨이 편하겠지 !
