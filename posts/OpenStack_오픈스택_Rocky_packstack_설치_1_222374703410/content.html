# 서론

Octavia 버전 오픈스택에서 로드밸런스 실습이 이루어 지지 않아

오픈스택 Rocky 버전으로 수동 설치 후 재시도.

그런데 여기서도 로드밸런스 서비스가 잘 이루어지지 않아(수동 설치 시, 각 프로그램들의 버전 차이가 원인인듯)

오픈스택 Rocky 버전 packstack 설치하여 실습.

1. 오픈스택 Rocky 버전,

Packstack 설치

(1) 호스트명 설정

[controller 노드] [compute 노드] [network 노드] 3대 다 설정.

​

(1) vi /etc/hosts 아래에 다음 내용 추가

컨트롤러노드-IP controller 컴퓨트노드-IP compute 네트워크노드-IP network # 예시: 10.10.10.1 controller

​

(2) vi /etc/hostname 파일에 기존 내용 지우고 다음과 같이 수정

controller ## 컨트롤러면 controller ## 컴퓨트면 compute ## 네트워크면 network 작성 !!!!

저장 및 종료.

​

init 6 로 재시작.

​

​

(2) 네트워크 설정

[controller 노드] [compute 노드] [network 노드] 3대 다 설정.

​

vi /etc/sysconfig/network-scripts/ifcfg-ens33
BOOTPROTO=static 으로 수정 ONBOOT=yes 로 수정 마지막 줄 밑에 다음 내용 추가 IPADDR=[IP주소] NETMASK=255.255.255.0 GATEWAY=[게이트웨이 주소] DNS1=8.8.8.8

저장 및 종료

systemctl restart network

​

​

(3) 방화벽 설정

[controller 노드] [compute 노드] [network 노드] 3대 다 설정.

systemctl stop firewalld systemctl disable firewalld systemctl stop NetworkManager systemctl disable NetworkManager setenforce 0 sed -i 's/=enforcing/=disabled/g' /etc/sysconfig/selinux

​

init 6 재시작

​

​

(4) ssh 설정

[controller 노드]

ssh-keygen 엔터 3번

엔터 3번.

​

ssh-copy-id root@controller 패스워드 입력 ssh-copy-id root@compute 패스워드 입력 ssh-copy-id root@network 패스워드 입력

​

(5) 오픈스택 설치 파일 다운

설치는 쭉 [controller 노드] 에서 진행.

​

yum update -y yum install centos-release-openstack-rocky -y yum install openstack-packstack -y yum install openstack-utils -y yum install chrony ntp -y

6) 오픈스택 설치 파일 설정

packstack --gen-answer-file=/root/rocky-answer.txt
vi /root/rocky-answer.txt
CONFIG_CONTROLLER_HOST=[컨트롤러 노드 IP] CONFIG_COMPUTE_HOSTS=[컴퓨트 노드 IP] CONFIG_NETWORK_HOSTS=[네트워크 노드 IP] CONFIG_PROVISION_DEMO=n CONFIG_NTP_SERVERS=0.centos.pool.ntp.org iburst, 1.centos.pool.ntp.org iburst, 2.centos.pool.ntp.org iburst, 3.centos.pool.ntp.org iburst CONFIG_CINDER_VOLUMES_SIZE=200G CONFIG_DEFAULT_PASSWORD= 비번 CONFIG_MARIADB_PW= 비번 CONFIG_LBAAS_INSTALL=y CONFIG_SWIFT_INSTALL=n CONFIG_HEAT_INSTALL=y CONFIG_STORAGE_HOST= 컴퓨트노드 IP

저장 및 종료

​

​

(6) 오픈스택 설치 진행

2. 초기 설정

1) 데시보드 세션 타임아웃 설정

[controller 노드]

vi /etc/openstack-dashboard/local_settings

위 파일에서 아래 세션타임아웃 부분을 찾아

SESSION_TIMEOUT = 1800

SESSION_TIMEOUT = 36000 이렇게 수정한다.

​

​

2) 네트워크 노드 네트워크 설정

[network 노드]

vi /etc/sysconfig/network-scripts/ifcfg-ens33 DEVICE=ens33 TYPE=OVSPort DEVICETYPE=ovs OVS_BRIDGE=br-ex ONBOOT=yes
vi /etc/sysconfig/network-scripts/ifcfg-ens33 DEVICE=br-ex DEVICETYPE=ovs TYPE=OVSBridge BOOTPROTO=static IPADDR=네크워크 노드 IP NETMASK=255.255.255.0 GATEWAY=게이트웨이 ONBOOT=yes
systemctl restart network

3) 노바 인스턴스 생성 타임아웃 설정

[compute 노드]

vi /etc/nova/nova.conf

위 파일에서 맨 앞 [DEFAULT] 섹션에 아래 내용을 추가한다.

instance_build_timeout = 3600 block_device_allocate_retries=1200 block_device_allocate_retries_interval=3

설정 저장 후, systemctl restart openstack-nova-탭으로 관련 프로그램들 재시작 해준다.

탭이 안되는 경우 아래 명령어로 찾아 재시작 해주세요.

systemctl list-unit-files | grep nova

나의 경우 아래 하나 나와서 재시작 해주었다.

systemctl restart openstack-nova-compute.service

* 4) swift 꺼두기

[controller 노드]

혹시 컨트롤러가 너무 느리다면..

팩스택으로 자동 설치할 때 깔린 swift 서비스를 꺼주자.

여기서는 swift 서비스를 사용하지 않을 것이기 때문에 아래 명령어로 관련 프로그램 다 꺼준다.

systemctl stop openstack-swift*
