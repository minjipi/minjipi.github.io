1. 도커가 뭐에요?

쉬운 설명

도커는 프로그램을 실행하는 데 필요한 모든 것 (코드, 라이브러리, 설정, 실행환경 등)을

하나의 박스(컨테이너)에 담고,

이 박스가 격리된 환경에서 실행될 수 있게 하여 어디서나 동일하게 실행할 수 있도록 해주는 기술입니다.

​

왜, ppt로 발표 할때 꼭 발표장에서 폰트가 깨지곤 하죠?

도커를 쓰면 이런 문제가 사라집니다.

노트북 + 파워포인트 설치 + 폰트까지 포함된 실행 환경 전체를 통째로 하나의 박스에 담아서, 어디서든 그대로 실행되게 만드는 기술이니까요.

​

내가 만든 발표 자료는 내 노트북뿐만 아니라 회사 PC, 클라우드 서버, 심지어 해외 개발자의 노트북에서도 똑같이 열리고 실행됩니다.

​

​

2. 도커 왜 써요?

도커가 좋은건 알겠는데,

왜 가상머신 안쓰고 도커를 쓸까요?

뭐가 다르길래?

​

가상머신과 도커의 차이

  • 가상머신(VM): 하이퍼바이저 위에 각기 다른 OS를 올리는 방식이라 무겁고, 부팅에도 시간이 걸립니다.

  • 도커(Docker): 운영체제를 새로 올리지 않고, 호스트 OS 커널을 공유하는 방식이라 훨씬 가볍고 실행이 빠릅니다.

​

가상머신과 도커의 차이

​

설치가 복잡한 소프트웨어도 명령어 한 줄 (docker run)이면 끝! 현업에서 개발-테스트-배포 과정이 단순해지고, 협업도 쉬워집니다.

​

여기서 한걸음 더 가봅시다.

운영체제를 새로 올리지 않고, 호스트 OS 커널을 공유하는 방식이 좀 더 궁금한가요? 사실 이게 그다지 신기술은 아닙니다.

​

​

3. 도커 기술에 대하여

개발자 면접 대비 단골 질문

도커는 리눅스 커널의 네임스페이스(namespace)와 cgroup(control group) 기능을 이용한 경량 가상화 기술입니다.

​

  • 네임스페이스: 프로세스를 격리된 공간에 실행시켜 각 컨테이너가 자기만의 파일시스템, 네트워크, PID를 가진 것처럼 보이게 한다.

  • cgroup: CPU, 메모리 같은 리소스를 프로세스 단위로 제한, 관리한다.

​

새로운 운영체제를 띄우는 게 아니라

하나의 리눅스 커널 위에서 여러 개의 격리된 실행환경을 만든 것 입니다. 그래서 도커가 가볍고 빠른 것이죠.

​

이 부분에 대해서는 이 포스팅에서

직접 리눅스 내부를 살펴보며 더 자세히 살펴봅니다. 😀

​

​

4. 정리

  • 도커가 뭐에요?

→ 리눅스 커널의 namespace, cgroups 기능을 활용한 경량 가상화 플랫폼

​

  • 도커를 왜 쓰나요?

→ 환경 일관성, 배포 표준화, 확장성, 경량성

​

  • 가상머신과 차이는 뭐에요?

→ VM은 하이퍼바이저 위에 OS를 올리지만,

도커는 호스트 커널을 공유. 그래서 가볍고 빠르다.
