[OpenStack]

오픈스택 수동설치

​

neutron(네트워크 서비스)

컨트롤러 노드 설치 및 설정

​

1) DB 생성

[1] controller 노드

데이터베이스 서버에 root 사용자로 연결.

mysql -u root -p

keystone 데이터베이스 생성

CREATE DATABASE neutron;

​

keystone 데이터베이스에 적합한 액세스를 부여하고, KEYSTONE_DBPASS 를 적절한 암호로 변경한다.

GRANT ALL PRIVILEGES ON neutron.* TO 'neutron'@'localhost' IDENTIFIED BY 'qwer1234';
GRANT ALL PRIVILEGES ON neutron.* TO 'neutron'@'%' IDENTIFIED BY 'qwer1234';

​

2) 사용자 설정 및 엔드포인트 생성

[1] controller

source ~/keystone_admin
openstack user create --domain default --password-prompt neutron

하고 패스워드 설정.

openstack role add --project service --user neutron admin
openstack service create --name neutron --description "OpenStack Networking" network
openstack endpoint create --region RegionOne network public http://컨트롤러 IP:9696 openstack endpoint create --region RegionOne network internal http://컨트롤러 IP:9696 openstack endpoint create --region RegionOne network admin http://컨트롤러 IP:9696

3) 서비스 설치

[1] controller

dnf --enablerepo=centos-openstack-victoria,powertools -y install openstack-neutron openstack-neutron-ml2

​

4) 서비스 설정

[1] controller

백업.

mv /etc/neutron/neutron.conf /etc/neutron/neutron.conf.org
vi /etc/neutron/neutron.conf

새로 생성한 파일에 아래 내용 추가. (주석해제 및 내용 추가, 변경)

​

[DEFAULT] core_plugin = ml2 service_plugins = router auth_strategy = keystone state_path = /var/lib/neutron dhcp_agent_notification = True allow_overlapping_ips = True notify_nova_on_port_status_changes = True notify_nova_on_port_data_changes = True transport_url = rabbit://openstack:qwer1234@컨트롤러 IP [keystone_authtoken] www_authenticate_uri = http://컨트롤러 IP:5000 auth_url = http://컨트롤러 IP:5000 memcached_servers = 컨트롤러 IP:11211 auth_type = password project_domain_name = default user_domain_name = default project_name = service username = neutron password = 비번 [database] connection = mysql+pymysql://neutron:비번@컨트롤러 IP/neutron [nova] auth_url = http://컨트롤러 IP:5000 auth_type = password project_domain_name = default user_domain_name = default region_name = RegionOne project_name = service username = nova password = 비번 [oslo_concurrency] lock_path = lock_path = /var/lib/neutron/tmp
chmod 640 /etc/neutron/neutron.conf chgrp neutron /etc/neutron/neutron.conf

​

파일 수정 및 내용 추가.

vi /etc/neutron/metadata_agent.ini

[DEFAULT] 섹션에 다음 추가

nova_metadata_host = 컨트롤러 IP metadata_proxy_shared_secret = 비번

​

219번 라인 쪽에 memcache_servers 부분 수정.

memcache_servers = 컨트롤러 IP:11211

​

vi /etc/neutron/plugins/ml2/ml2_conf.ini

파일 마지막 줄에 다음을 추가.

[ml2] type_drivers = flat,vlan,gre,vxlan tenant_network_types = mechanism_drivers = openvswitch extension_drivers = port_security

​

vi /etc/nova/nova.conf

맨 앞 [DEFAULT] 섹션에 다음을 추가.

use_neutron = True linuxnet_interface_driver = nova.network.linux_net.LinuxOVSInterfaceDriver firewall_driver = nova.virt.firewall.NoopFirewallDriver

​

[neutron] 섹션에 다음을 추가.

auth_url = http://컨트롤러 IP:5000 auth_type = password project_domain_name = default user_domain_name = default region_name = RegionOne project_name = service username = neutron password = 비번 service_metadata_proxy = True metadata_proxy_shared_secret = 비번

저장 후 파일 나가기 (:wq)

​

ln -s /etc/neutron/plugins/ml2/ml2_conf.ini /etc/neutron/plugin.ini
su -s /bin/bash neutron -c "neutron-db-manage --config-file /etc/neutron/neutron.conf --config-file /etc/neutron/plugin.ini upgrade head"
systemctl enable neutron-server neutron-metadata-agent systemctl restart neutron-server neutron-metadata-agent
systemctl restart openstack-nova-api
