[OpenStack]

오픈스택 수동설치

Keystone 설치(인증 서비스)

​

1) DB 생성

[1] controller 노드

데이터베이스 서버에 root 사용자로 연결.

mysql -u root -p

keystone 데이터베이스 생성

CREATE DATABASE keystone;

​

keystone 데이터베이스에 적합한 액세스를 부여하고, KEYSTONE_DBPASS 를 적절한 암호로 변경한다.

GRANT ALL PRIVILEGES ON keystone.* TO 'keystone'@'localhost' IDENTIFIED BY 'qwer1234';
GRANT ALL PRIVILEGES ON keystone.* TO 'keystone'@'%' IDENTIFIED BY 'qwer1234';

​

​

2) 패키지 설치

[1] controller 노드

dnf --enablerepo=centos-openstack-victoria,powertools -y install openstack-keystone python3-openstackclient httpd mod_ssl python3-mod_wsgi python3-oauth2client

​

3) 설정 파일 수정

[1] controller 노드

vi /etc/keystone/keystone.conf

위 파일에서 몇 가지 수정한다.

​

(1)

440번 라인 쪽을.. 다음과 같이 수정. 주석처리 되어 있으면 주석도 해제!

memcache_servers = 192.168.1.101:11211

(vi 편집기에서 /memcache_servers 이렇게 찾으세요)

​

(2)

615번 라인 쪽에

connection = mysql+pymysql://keystone:qwer1234@컨트롤러 IP/keystone

​

(3)

2499번 라인 쪽에

provider = fernet

주석 해제 확인해 주세요.

​

​

4) DB에 설정 내용 저장

su -s /bin/bash keystone -c "keystone-manage db_sync"

​

5) 키스톤 설정

Fernet 키를 초기화한다.

keystone-manage fernet_setup --keystone-user keystone --keystone-group keystone
keystone-manage credential_setup --keystone-user keystone --keystone-group keystone

​

​

6) 부트스트랩 설정

keystone-manage bootstrap --bootstrap-password qwer1234 \ 설정할 비밀번호☝🏻 --bootstrap-admin-url http://컨트롤러 IP:5000/v3/ \ --bootstrap-internal-url http://컨트롤러 IP:5000/v3/ \ --bootstrap-public-url http://컨트롤러 IP:5000/v3/ \ --bootstrap-region-id RegionOne

​

7) 웹 서버 설정

vi /etc/httpd/conf/httpd.conf

위 파일에서 ServerName 부분을 찾아 아래처럼 수정.

ServerName 192.168.x.x 컨트롤러IP

( /ServerName 이렇게 찾아서 수정해 주세요.)

​

ln -s /usr/share/keystone/wsgi-keystone.conf /etc/httpd/conf.d/
systemctl enable httpd.service systemctl restart httpd.service

​

8) 확인

keystone_admin 파일 새로 생성

vi ~/keystone_admin
export OS_USERNAME=admin export OS_PASSWORD=설정할 비밀번호 export OS_PROJECT_NAME=admin export OS_USER_DOMAIN_NAME=Default export OS_PROJECT_DOMAIN_NAME=Default export OS_AUTH_URL=http://컨트롤러 IP:5000/v3 export OS_IDENTITY_API_VERSION=3
source ~/keystone_admin

방금 생성한 파일 적용하고..

​

이제 아래의 openstack 명령어가 실행만 되면 된다.

'키스톤'은 오픈스택 명령어를 사용할 수 있도록 인증해주는 인증 서비스.

openstack domain list openstack project list openstack user list

* openstack domain list 에러

Internal Server Error (HTTP 500)

​

💡해결 :: httpd restart 먼저 해보고, 나의 경우 위 설정에 오타가 있거나 뭔가 실수한 부분이 있어서 해당 에러가 발생했다. Internal Server Error (HTTP 500) 뜬다면 DB에 키스톤부터 지우고, 거기서부터 다시 찬찬히 해보시길 바랍니다ㅠ 다시 할 때 패키지 설치는 할 필요 없고, 오타난 부분이 있는지 주석처리는 다 해제해 줬는지 체크해가며 다시 해 보세요 !!!

​

​

9) 서비스 프로젝트 생성

openstack project create --domain default --description "Service Project" service
openstack project list
