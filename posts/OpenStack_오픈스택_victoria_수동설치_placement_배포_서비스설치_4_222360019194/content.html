오픈스택 수동설치

​

placement (배포 서비스) 설치

​

* Stein 버전 이전에는 nova 서비스에 포함되어 있었지만, stein 이후 버전에서는 따로 빠져나옴.

​

​

1) placement DB 생성

[1] controller 노드

데이터베이스 서버에 root 사용자로 연결.

mysql -u root -p

placement 데이터베이스 생성

CREATE DATABASE placement; GRANT ALL PRIVILEGES ON placement.* TO 'placement'@'localhost' IDENTIFIED BY 'qwer1234'; GRANT ALL PRIVILEGES ON placement.* TO 'placement'@'%' IDENTIFIED BY 'qwer1234';

​

​

2) 사용자 설정 및 엔드포인트 생성

[1] controller

source ~/keystone_admin
openstack user create --domain default --password-prompt placement

하고 패스워드 설정.

openstack role add --project service --user placement admin
openstack service create --name placement --description "OpenStack Image" image
openstack endpoint create --region RegionOne placement public http://컨트롤러 IP:8778 openstack endpoint create --region RegionOne placement internal http://컨트롤러 IP:8778 openstack endpoint create --region RegionOne placement admin http://컨트롤러 IP:8778

​

3) 서비스 설치

[1] controller

dnf --enablerepo=centos-openstack-victoria,powertools -y install openstack-placement-api

​

4) 서비스 설정

[1] controller

placement.conf 파일 백업.

mv /etc/placement/placement.conf /etc/placement/placement.conf.org

​

새로 생성하고,

vi /etc/placement/placement.conf

아래 내용 추가. (주석해제 및 내용 추가, 변경)

​

[DEFAULT] debug = false [api] auth_strategy = keystone [keystone_authtoken] www_authenticate_uri = http://컨트롤러 IP:5000 auth_url = http://컨트롤러 IP:5000 memcached_servers = 컨트롤러 IP:11211 auth_type = password project_domain_name = default user_domain_name = default project_name = service username = placement password = 설정한 비밀번호 [placement_database] connection = mysql+pymysql://placement:설정한 비밀번호@컨트롤러 IP/placement
chmod 640 /etc/placement/placement.conf chgrp placement /etc/placement/placement.conf

​

아래 파일 수정.

vi /etc/httpd/conf.d/00-placement-api.conf

15번째 라인 ..

<Directory /usr/bin> Require all granted </Directory>

추가.

​

5) DB에 설정 내용 저장

su -s /bin/sh -c "placement-manage db sync" placement

웹 서버 재시작.

systemctl restart httpd

​
