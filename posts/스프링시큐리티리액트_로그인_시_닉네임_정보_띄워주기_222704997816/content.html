어떻게 구현을 해야하나 고민했었는데 결국 jwt decode가 답이었다.

​

1) 스프링 시큐리티 Jwt 토큰 발급해주는 부분에서

public class JwtTokenUtil implements Serializable { [ 생략 ] private String doGenerateToken(UserDetails userDetails) { UserLoginRes userLoginRes= (UserLoginRes)userDetails; Map<String, Object> idx = new HashMap<>(); Map<String, Object> nickname = new HashMap<>(); nickname.put("idx",userLoginRes.getIdx()); nickname.put("nickname",userLoginRes.getNickname()); return Jwts.builder().setClaims(idx).setClaims(nickname).setIssuedAt(new Date(System.currentTimeMillis())) .setExpiration(new Date(System.currentTimeMillis() + JWT_TOKEN_VALIDITY * 1000)) .signWith(SignatureAlgorithm.HS512, secret).compact(); } }

idx와 nickname 값을 추가해준다.

(idx는 추후 좋아요 구현 시 필요해서 추가함. 포스팅 핵심인 닉네임 정보와는 무관함)

​

그리고 발급된 jwt 토큰을 확인해보자.

토큰 decode는 https://jwt.io/ 사이트에서 확인했다.

​

​

발급한 idx와 nickname 값이 제대로 담겨있다.

이제 이것을 페이지에 띄워주자.
