​

사이드카 패턴(Sidecar) 이란?

사이드카 패턴은

원래 사용하려고 했던 기본 컨테이너의 기능을 확장하거나 강화하는 용도의 컨테이너를 추가하는 패턴이다.

​

기본 컨테이너와 독립적으로 동작하는 별도의 컨테이너를 붙이는 패턴이기 때문에 어플리케이션 컨테이너의 변경이나 수정 없이 독립적으로 동작하는 컨테이너를 붙였다 뗐다 할 수 있다.

​

​

​

오! 나의 여신님에서 케이와 베르단디가 타고 다니는 BMW 오스카 리브맨 스페셜과, 옆에 붙은 사이드카를 붙이든 떼어내든 상관 없이 오토바이는 동일하게 운전할 수 있는 것처럼.

​

​

사이드카 패턴을 이용해,

git에 올려둔 웹사이트 코드를 자동으로 받아오는 컨테이너와

받아온 것을 웹서버에 뿌려주는 컨테이너가 있는

아래 그림과 같은 하나의 파드를 생성해 보도록 한다.

하나는 서비스 하는 컨테이너, 하나는 받아오는 컨테이너.

​

사이드카 패턴(Sidecar) 예제

(1)

우선 사이드카 디렉토리를 생성한다.

mkdir /sidecar cd /sidecar

​

(2)

/sidecar 안에 아래의 [ Dockerfile ], [ contents-pull.sh ] 두 파일을 생성한다.

[ Dockerfile ]

FROM ubuntu:16.04 RUN apt-get update && apt-get install -y git ADD ./contents-pull.sh /contents-pull.sh RUN chmod 755 /contents-pull.sh WORKDIR / cmd /contents-pull.sh

​

[ contents-pull.sh ]

#!/bin/bash if [ -z $CONTENTS_SOURCE_URL ]; then exit 1 fi git clone $CONTENTS_SOURCE_URL /data cd /data while true do date sleep 60 git pull done

​

$CONTENTS_SOURCE_URL 은 환경변수로 만들어 두어,

이따 컨테이너 생성 시 환경변수 값을 전달해 줄 예정이다.

웹서버가 뿌려줄 깃 페이지 주소가 들어간다.

​

​

(3)

도커 허브에 sidecar 레포지터리 만들고,

​

​

(4)

이미지 생성

docker build --tag ghdalswl77/coffee-front-puller:1.0 ./

​

(5)

환경변수 옵션 지정하는 옵션을 줘서, 컨테이너 생성

docker run -e CONTENTS_SOURCE_URL=https://github.com/minjipi/web-front --name test -d ghdalswl77/coffee-front-puller:1.0

​

(6)

레포지터리에 이미지 업로드

docker push ghdalswl77/coffee-front-puller:1.0

​

(7) yaml 파일 생성

​

[sidecar.yaml]

apiVersion: v1 kind: Pod metadata: name: sidecar spec: containers: - name: front-server image: nginx volumeMounts: - mountPath: /usr/share/nginx/html name: contents-vol - name: puller image: ghdalswl77/coffee-front-puller:1.0 # image: [도커허브ID]/[레포지터리]:[버전] env: - name: CONTENTS_SOURCE_URL value: "https://github.com/minjipi/web-front" volumeMounts: - mountPath: /data name: contents-vol volumes: - name: contents-vol emptyDir: {}

​

이 yaml 파일을 살펴보자.

우선 아래의 그림과 같이

두 개의 컨테이너가 '하나의 볼륨'을 사용하게 해주었다.

​

그리고 mountPath 에 각각 위와 같이 작성하여

하나의 하드디스크를 같이 쓰게 해 준다.

​

​

puller 컨테이너는 컨텐츠를 1분에 한 번씩 받아오고,

nginx 컨테이너는 컨텐츠를 사용자들이 요청하면 전송해준다.

​

(8)

위 파일들로- pod 생성

kubectl apply -f [파일명]

​

​

(9)

생성한 pod IP 확인:

kubectl get pods -o wide

​

잘 됐나~ 접속해본다.

curl http://사이드카-IP
