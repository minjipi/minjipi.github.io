3) 도커 파일

도커 파일이란, 컨테이너를 만들고 해야하는 일련의 작업들을 미리 선언함으로써 매번 해당 작업을 하지않고도 컨테이너 생성 시 자동으로 등록된 작업이 실행된 후 컨테이너를 생성할 수 있는 파일이다.

​

매뉴얼 작업을 기록한 Dockerfile 생성 (파일의 이름이 Dockerfile) 하고,

빌드 명령어가 Dockerfile 을 읽어 이미지를 생성하는 과정이다.

​

3. 이미지 빌드 명령어

이미지를 빌드하는 명령어는 다음과 같다.

docker build [OPTIONS] PATH | URL | docker build --tag 이미지이름:버전 [도커 파일이 있는 위치] 입력 안해도 되는 것.
docker build --tag message:1.0 ./

* 그래서 Dockerfile 이 내 컴퓨터에 저장되어 있는게 아니라 다른 사람 컴퓨터에 저장되어 있고 URL로 접근이 가능하면 /a 라는 디렉토리 대신에 URL을 입력하는 것. 예를 들어, 내가 내 웹서버에 Dockerfile을 올려두면 거기서 Dockerfile로 접속할 수 있는 URL을 입력하면 된다는 말. 경로 대신에 URL도 쓸 수 있다.

# 도커파일로 톰캣 실행

1. 디렉터리에 Dockerfile, test.jsp 파일 생성.

​

Dockerfile 파일 내용:

FROM tomcat:latest RUN mkdir /usr/local/tomcat/webapps/만들어줄-디렉터리-이름 ADD ./test.jsp /usr/local/tomcat/webapps/만들어줄-디렉터리-이름/test.jsp EXPOSE 8080 CMD ["/usr/local/tomcat/bin/catalina.sh", "run"]

​

test.jsp 파일 내용:

<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8" %> <%@ page import="java.net.InetAddress" %> 클라이언트 IP <%=request.getRemoteAddr()%><br> 요청URI <%=request.getRequestURI()%><br> 요청URL: <%=request.getRequestURL()%><br> 서버이름 <%=request.getServerName()%><br> <% InetAddress inet= InetAddress.getLocalHost(); %> 동작 서버 IP <%=inet.getHostAddress()%><br> 서버포트 <%=request.getServerPort()%><br>

​

이미지 빌드

docker build --tag custom_tomcat:1.0 ./

​

이미지 실행

docker run -it -p 8080:8080 --name custom_tomcat1 custom_tomcat:1.0

​

접속

본체IP:8080/만들어줄-디렉터리-이름/test.jsp
