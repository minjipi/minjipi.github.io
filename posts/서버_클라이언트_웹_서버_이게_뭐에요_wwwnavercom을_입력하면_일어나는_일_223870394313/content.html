서버, 클라이언트, 웹 서버.

에 대해 설명해 주세요.

이 질문에 가장 명확하게 답하는 방법은,

우리가 실제 웹사이트에 접속할 때 어떤 일이 일어나는지를 따라가보는 것입니다.

예를 들어, 브라우저에 www.naver.com을 입력하면

무슨 일이 벌어질까요?

​

인터넷 너머 어떤 통신이 오가는지 단계별로 살펴보면,

서버, 클라이언트, 웹 서버의 역할이

자연스럽게 드러납니다.

​

네이버 홈페이지를 보기 까지의 과정

‘www.naver.com’을 입력했을 때 일어나는 일들

  1. 브라우저가 요청(Request)을 보냅니다.

  2. 이 요청은 먼저 웹 서버에 도착해요.

  3. 웹 서버는 단순한 HTML, CSS 같은 정적 파일을 전달하기도 하고,

  4. 경우에 따라 요청을 백엔드 서버로 넘겨요.

  5. 백엔드 서버는 필요한 경우 데이터베이스(DB)와 통신해

  6. 동적으로 데이터를 가져와 응답을 만들어냅니다.

  7. 응답 결과는 다시 클라이언트(브라우저)로 전달되고,

  8. 브라우저는 이 파일들과 데이터를 받아 화면에 보여줍니다.

​

이 흐름은 꼭 알고 있어야 하는데요.

이제 좀 더 자세히 알아봅시다.

먼저 1번, 요청을 보내는 쪽부터 살펴볼게요.

​

1. 클라이언트

클라이언트는 서버에게 요청하는 프로그램 입니다.

쉽게 말해, 서비스를 '요청하는 쪽'이에요.

​

  • 우리가 크롬이나 사파리 같은 브라우저에서 www.naver.com에 접속하면

  • → 이때 브라우저가 클라이언트입니다.

  • 모바일 앱에서 쇼핑몰 상품 목록을 불러올 때

  • → 그 앱도 클라이언트가 됩니다.

  • 터미널에서 curl 명령어로 웹 요청을 보낼 때

  • → curl도 클라이언트 역할을 해요.

즉, 클라이언트는 서버에 요청을 보내고, 그 응답을 받아서 화면에 보여주는 역할을 담당합니다.

​

​

2. 서버

클라이언트가 요청을 보내면,

그 요청을 받아서 처리해주는 컴퓨터가 바로 #서버 (Server)예요.

서버는 단순한 컴퓨터 그 자체라기보다,

특정한 서비스를 제공하는 역할을 맡은 프로그램이나 시스템을 말해요.

​

예를 들어:

  • 어떤 서버는 웹페이지를 보여주는 역할 (→ 웹 서버)

  • 어떤 서버는 데이터베이스에서 정보를 저장하고 꺼내오는 역할 (→ 데이터베이스 서버)

  • 어떤 서버는 사용자 인증만 해주는 역할 (→ 인증 서버)

즉, ‘서버’는 어떤 요청을 받아서 적절한 응답을 보내주는 프로그램 혹은 컴퓨터라고 이해하면 됩니다.

​

​

3. 웹 서버

웹서버는 뭘까요?

바로 요청을 받아 파일을 전달하는 프로그램 입니다.

​

#웹서버 (Web Server)는

웹 브라우저(클라이언트)로부터 요청을 받아,

HTML, CSS, JavaScript 같은 웹페이지 구성 파일들을 돌려주는 역할을 합니다.

​

예를 들어 내가 네이버 메인 페이지를 보고 싶다고 요청을 하면,

웹 서버는 index.html, style.css, main.js 같은 파일을 응답으로 보내주는 거예요.

이런 파일들은 우리가 흔히 말하는 정적(static) 파일이고,

웹 서버는 이 정적 파일을 빠르고 안정적으로 전송해주는 역할을 합니다.

네이버 메인 페이지에서 개발자도구 -> 네트워크 탭에서 확인할 수 있습니다.

대표적인 웹 서버 프로그램으로는 Nginx, Apache HTTP Server 등이 있어요.

​

​

4. HTML과 자바스크립트가 브라우저에서 실행된다.

클라이언트는 웹 서버로부터 전달받은 HTML 파일을 웹 브라우저에서 열게 됩니다.

이때 HTML 내부에 포함되어 있는 자바스크립트 코드도 함께 실행돼요.

자바스크립트는 단순히 화면에 글자를 바꾸는 기능뿐 아니라, 필요한 데이터를 서버에 요청해 받아오는 역할도 할 수 있어요.

​

예를 들어, 자바스크립트 안에 포함된 Axios 같은 코드가 실행되면,

서버에 “이 뉴스 목록을 주세요”와 같은 요청을 보냅니다.

Axios는 자바스크립트에서 HTTP 요청을 보내기 위해 자주 쓰이는 라이브러리입니다.

브라우저에서 서버로 데이터를 요청하거나, 응답을 받아오는 작업을 쉽게 해줘요.

그러면 웹 서버가 다시 그 요청을 받아,

필요한 리소스를 응답으로 돌려주게 되는 거죠.

​

​

5. 백엔드 서버는 데이터를 처리하는 역할

웹 서버는 단순히 정적인 파일을 전달할 뿐이지만,

백엔드 서버는 그게 아니에요.

자바스크립트에서 예를 들어 Axios 같은 라이브러리를 사용해

“뉴스 목록 데이터를 주세요” 같은 요청을 보낼 때,

이 요청을 받아 처리하는 게 백엔드 서버예요.

백엔드 서버는 주로 Node.js, Spring, Django 같은 프레임워크로 구성돼 있고,

이 서버는 내부적으로 데이터베이스(DB)와 연결되어 있어요.

​

​

6. 백엔드 DB에서 데이터를 꺼내와 화면을 채운다

사용자가 보는 뉴스 목록, 게시글 제목, 프로필 이미지 경로 같은 정보들은

보통 전부 백엔드 서버 뒤에 있는 데이터베이스(DB)에 저장돼 있어요.

자바스크립트는 Axios 같은 도구를 이용해서 백엔드 서버에 데이터를 요청하고,

백엔드는 그 요청을 받아 DB에서 데이터를 꺼낸 다음,

JSON 같은 포맷으로 클라이언트에게 돌려줍니다.

브라우저는 이 데이터를 받아 화면에 뿌려줘요.

예를 들어, '상품 목록’을 보여주는 페이지라면,

상품 이름, 이미지 URL, 가격 등의 정보

모두 백엔드 DB에서 끌어온 데이터인 거죠.

​

​

더 깊이 들어가 이야기할 수 있는 부분이 많지만,

이번 글에서는 어디까지나

서버, 클라이언트, 웹 서버라는 핵심 개념을 명확히 이해하는 데 초점을 맞추려 합니다.

​

서버가 뭔지, 클라이언트는 어떤 역할을 하는지, 웹 서버는 어디에 위치하는지

그 구조와 역할이 훨씬 자연스럽게 정리될 거예요.
