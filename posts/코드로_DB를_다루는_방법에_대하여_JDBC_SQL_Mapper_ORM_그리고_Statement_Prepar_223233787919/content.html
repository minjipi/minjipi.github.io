객체 지향 데이터베이스(object oriented database)가 있다. // (이건 이번 포스팅에서 자세히 설명 X)

​

​

1. 데이터베이스의 종류

1) 관계형 데이터베이스(relational database)

2. 객체 지향 데이터베이스(object oriented database) (이번 포스팅에서 설명 X)

​

관계형 데이터 베이스는 테이블과 열 로 구성된다.

SQL 을 사용하여 데이터베이스 스키마 생성, 자료의 검색·관리·수정, 데이터베이스 객체에 접근·관리한다.

​

2. 관계형 데이터베이스의 사용

1) SQL(Structured Query Language)

SQL(Structured 은 관계형 데이터베이스 관리 시스템(DBMS) 에서 데이터베이스 스키마 생성, 자료의 검색, 관리, 수정, 그리고 데이터베이스 객체 접근 관리 등을 위해 고안된 언어이다. 가장 대표적으로, 자바에서도 SQL로 작성된 간단한 데이터베이스 처리 명령어를 DBMS에게 보내어 데이터베이스 처리를 지시한다.

​

2) JDBC

JDBC 는 자바에서 관계형 데이터베이스에 저장된 데이터를 접근 및 조작할 수 있게 하는 API이다.

JDBC는 다양한 DB 드라이버를 제공하기 때문에, 자바 개발자는 DBMS의 종류에 관계없이 JDBC API만을 이용하면 된다.

이미지 출처 : https://mysterlee.tistory.com/49

a. JDBC를 이용한 방법

SQL을 수동으로 작성해서,

SQL을 실행하고 실행된 결과를 객체에 수동으로 개발자가 저장.

JDBC(Java Database Connectivity) API 인 Statement 클래스와 ResultSet 클래스에 대해서 가볍게 알아보자.

​

자바에서 SQL문을 실행하 기 위해서는 Statement 클래스를 이용하고,

SQL문 실행 결과를 얻어오기 위해서 ResultSet 클래스를 이용한다.

​

JDBC(Java Database Connectivity)의 동작 순서는 다음과 같다.

JDBC Driver 로드 -> DB 연결 -> Statement 생성 -> SQL문 전송 -> 결과 리턴 ResultSet -> 연결 종료

i) Statement 클래스에서 자주 사용되는 메소드

ResultSet executeQuery(String sql)

주어진 sql문을 실행하고 결과는 ResultSet 객체에 반환

int executeUpdate(String sql)

INSERT, UPDATE, 또는 DELETE과 같은 sql문을 실행하고, sql 문 실행으로 영향을 받은 행의 개수나 0을 반환

void close()

Statement 객체의 데이터베이스와 JDBC 리소스를 즉시 반환

​

데이터를 검색하기 위해서는 executeQuery() 메소드를 사용하고, 추가, 수정, 삭제 와 같이 데이터 변경은 executeUpdate() 메소드를 이용한다. ResultSet 객체는 현재 데이터의 행(레코드 위치)을 가리키는 커서(cursor)를 관리한다. 초기 값은 첫 번째 행 이전을 가리키도록 되어있다. 따라서 ResultSet 클래스는 주로 커서의 위치와 관련된 메소드와 레코드를 가져오는 메소드를 제공한다.

​

ii) ResultSet 클래스에서 자주 사용되는 메소드

boolean first()

커서를 첫 번째 행으로 이동

boolean next()

커서를 다음 행으로 이동

boolean isFirst()

첫 번째 행이면 true 반환

​

예시로, 테이블의 모든 데이터 검색 student 테이블의 모든 데이터를 검색하는 코드는 다음과 같다.

Statement stmt = conn.createStatement(); ResultSet rs = stmt.executeQuery("select * from student");

​

검색된 데이터의 사용 검색의 결과는 ResultSet 객체에 저장된다.

ResultSet 클래스의 메소드를 이용하여 커서가 가리키는 현재 행에 대해 열의 값을 읽어 온다. 예를 들어 ResultSet 클래스의 객체 rs에 저장된 name 필드의 값과 id 필드의 값을 얻어내기 위해서는 각각 다음과 같이 한다.

String sName = rs.getString("name"); // name 필드 값 읽기 String id = rs.getString("id"); // id 필드 값 읽기

​

다음 코드는 ResultSet 객체에 저장된 각 행의 모든 열을 순차적으로 출력하는 코드이다.

while (rs.next()) { // rs에 저장된 다음 행으로 커서를 옮긴다. System.out.println(rs.getString("name")); System.out.println(rs.getString("id")); System.out.println(rs.getString("dept")); }

열의 값을 읽기 위해서는 열의 데이터 타입을 알고 있어야 하며, 그에 맞는 ResultSet 클래스의 메소드를 호출해야 한다. 또는 모든 데이터 타입에 대해 getString() 메소드로 읽을 수 있는데 이 경우는 모든 값이 문자열로 반환되므로 프 로그램 내에서 적절한 데이터 타입으로 변환해서 사용해야 한다.

이렇게 저장된 ResultSet 이 첫번째로 가리키는게 뭘까요?

당연히 첫번째 데이터인줄 알았다. 하지만 처음에는 첫번째 '전' 데이터를 가리키고 있다.

그렇기 때문에 next() 를 사용해서 커서를 다음 행으로 이동해야 첫번째 데이터에 접근하는 것이다.

1. Statement 클래스

Statement 클래스는 정적인 SQL 쿼리를 실행할 때 사용한다.

SQL 쿼리는 문자열로 작성되며, 쿼리의 내용이 동적으로 변경되지 않는 한 같은 쿼리를 여러 번 실행할 때 유용하다.

Statement statement = connection.createStatement(); String sql = "SELECT * FROM TableName WHERE column = 'value'"; ResultSet resultSet = statement.executeQuery(sql);

그러나 보안 측면에서 취약하며, SQL 쿼리를 작성할 때 문자열 연결을 사용하므로 SQL Injection 공격에 노출될 수 있다. 또 쿼리를 실행할 때마다 컴파일이 발생하므로 실행 속도가 느릴 수 있다.

​

​

2. PreparedStatement 클래스

PreparedStatement 클래스는 동적으로 변하는 매개변수를 갖는 SQL 쿼리를 실행할 때 사용한다.

SQL 쿼리에 매개변수를 사용하여 동적으로 쿼리를 생성할 수 있다.

PreparedStatement preparedStatement = connection.prepareStatement("SELECT * FROM TableName WHERE column = ?"); preparedStatement.setString(1, "value"); // 첫 번째 파라미터에 문자열 값 설정 ResultSet resultSet = preparedStatement.executeQuery(); // 쿼리 실행

PreparedStatement 클래스는 위와 같이 매개변수를 사용하여 SQL 쿼리를 동적으로 생성하므로, 쿼리의 재사용 및 성능 향상에 용이하며, SQL Injection을 방지할 수 있다. 또 미리 컴파일된 쿼리를 사용하므로 실행 속도가 빠르다.

​

​

​

하지만..

JDBC만을 이용하여 코딩하게 되면 위와 같은 작업을 직접 해주어야 하기 때문에 중복된 코드, 예외 처리 등 불편한 점이 많다.

b. SQL Mapper를 이용한 방법

SQL을 수동으로 작성해서 SQL을 실행하고

실행된 결과를 객체에 매퍼를 통해서 매핑

SQL Mapper 는개발자가 직접 작성한 SQL 문과 객체의 필드를 매핑하여 데이터를 객체화 해주는 프레임 워크이다. Java JDBC의 경우 각 클래스마다 쿼리문을 작성하였지만, Mapper 방식을 사용한다면 중복되는 코드들을 줄일 수 있다.

하지만 SQL Mapper 또한 개발자가 직접 다뤄야 하기 때문에 불편한 점이 있다. DBMS에 따라서 문법이 다르기 때문에 DBMS에 종속적이다. SQL Mapper에는 Mybatis 와 Spring JDBC가 있다.

​

MyBatis 는 JDBC를 단순화하여 SQL을 xml 파일에 작성해놓고 코드와 SQL을 분리하여 사용하는 방식이다. 따라서 소스코드에 SQL을 작성하지 않기 때문에 코드의 복잡성을 줄일 수 있다.

​

Spring JDBC 는 JDBC template에 DataSource, 즉 커넥션을 위한 설정들을 주입시켜주고 template을 통해 데이터를 꺼내오면서 한 단계 더 추상화했다. 이로 인해 Spring JDBC를 사용하면, 개발자가 연결, statement 준비와 실행, 트랜잭션 제어, 예외 처리 등을 따로 신경쓰지 않아도 된다.^^

​

그래도 어쨌든, SQL문을 직접 작성하면 SQL에 의존적인 개발을 하게 된다는 문제점이 있다.

​

c. ORM을 이용한 방법

자바코드로 SQL을 자동 생성해서 SQL을 실행하고

실행된 결과를 객체에 매핑

또 작성하는 코드는 객체지향이고, 관계형 데이터베이스는 데이터 중심의 구조를 갖고 있기 때문에 추상화, 상속, 다형성같은 개념을 갖는 객체를 데이터베이스에 직접 저장, 조회하기에는 어려움이 있다.

이렇게 각각 지향하는 목적이 다르기 때문에 패러다임 불일치가 발생하여 ORM이 나오게 된다.

​

ORM(Object Relational Mapping)은 말 그대로 객체와 관계형 DB를 매핑하는 것이다.

ORM은 객체간의 관계를 바탕으로 SQL문을 자동으로 생성하고, 직관적인 코드로 데이터 조작이 가능하다.
