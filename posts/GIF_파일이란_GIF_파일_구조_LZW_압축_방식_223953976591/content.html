1. GIF 파일이란?

Graphics Interchange Format.

8-bit 컬러 무손실 이미지 포맷

GIF 파일은 인터넷 전송과 온라인 콘텐츠 표현에

최적화된 형식으로 설계된 이미지 포맷입니다.

​

GIF 파일의 주요 특징은 다음과 같습니다.

​

  • 8비트(최대 256색) 팔레트 기반 이미지

  • 무손실 압축 (LZW 알고리즘)

  • 단일 파일 내 복수 이미지 프레임 -> 애니메이션 효과!

​

​

3. GIF 데이터 압축 방식 (LZW)

Lempel-Ziv-Welch 알고리즘

GIF에서 사용되는 LZW 압축은 Lempel-Ziv-Welch 알고리즘에 기반한 무손실 압축 기법입니다.

​

중복이 많은 데이터에 대해 높은 압축률을 달성하고,

GIF에서는 이미지의 색상 인덱스를

효율적으로 저장하기 위해 이 방식을 사용합니다.

​

압축의 핵심은 중복되는 시퀀스를 짧은 코드로 치환하는 것인데요. 간단한 예제 흐름을 통해 알아봅시다.

​

​

입력 시퀀스:

A B A B A B A B

1) 압축 과정 설명에 앞서 사전 규칙(핵심 개념)

  • STRING: 지금까지 읽은 문자열 시퀀스. 초기에는 1글자.

  • CHAR: 새로 읽은 문자 (1바이트)

  • STRING + CHAR: 새로운 후보 시퀀스 -> 테이블에 있으면 확장, 없으면 등록

​

2) 압축 과정

1. 초기화:

테이블에 0~255 코드 초기화 + Clear Code(256), End of Info Code(257)

2. STRING ← 첫 문자

3. 반복:

  1. 다음 문자 CHAR를 읽어 STRING+CHAR 탐색

  2. 존재하면 STRING ← STRING+CHAR (계속 시퀀스 확장)

  3. 없다면?

  4. STRING에 해당하는 코드 출력

  5. STRING+CHAR을 테이블에 등록 (새 코드 부여)

  6. STRING ← CHAR

4. 종료 시:

  • 마지막 STRING의 코드 출력

  • End of Info 코드 출력

4. 데이터 저장 방식

GIF 내부에서 이미지 정보는 이렇게 저장됩니다.

[시작 코드 크기] [255바이트 이하의 서브 블록들] [종료(0x00)]

  • LZW 압축 결과가 여러 개의 블록으로 쪼개짐

  • 각 블록은 [길이][데이터] 형식

  • 마지막에 0x00으로 마무리

​

디코더는 이 블록들을 읽으며 압축을 풀고,

다시 원래 색상 인덱스들을 복원합니다.
