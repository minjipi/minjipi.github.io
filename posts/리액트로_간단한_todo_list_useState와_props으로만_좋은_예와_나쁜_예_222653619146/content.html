리액트로 todo list를 만드는데

안좋은 예와 괜찮은 예가 만들어져서..

정리할 겸 올린다.

​

리액트로 구현하는 TodoList 중 좋은 예제는 아니며, useState와 props 으로만 최대한 간단하게 구현해보는 것이 목표였다.

​

​

[안좋은 예]

우선 이 코드가 안좋은 예라고 생각한 이유는 리액트의 기능을 잘 활용하지 못한 듯 해서다.

function Todo() { const [word, setWord] = useState(""); const [todoList, setTodoList] = useState([]); const [cnt, setCnt] = useState(1); const todoCheck = (id) => { setTodoList( todoList.map((todo) => todo.id === id ? { id: todo.id, contents: todo.contents, check: true } : todo ) ); }; return ( <> <Input type="text" value={word} onChange={(e) => { setWord(e.target.value); }} ></Input> <Btn type="button" onClick={(e) => { const todo = { id: cnt, contents: word, check: false }; setCnt(cnt + 1); setTodoList([...todoList, todo]); setWord(""); }} > 추가 </Btn> {todoList && todoList.map((todo, index) => ( <> <p className={todo.check ? "true" : ""} isChecked key={todo.id} onClick={(e) => { todoCheck(todo.id); }} > {index + 1}: {todo.contents} {todo.check ? ( "" ) : ( <DeleteBtn type="button" onClick={(e) => { setTodoList(todoList.filter((item) => todo.id !== item.id)); }} > 🗑 </DeleteBtn> )} </p> </> ))} </> ); }

​

1. todoCheck 기능

리스트에 해결 한 항목을 체크하는 기능이다.

Ptag 태그를 누르면,

const todoCheck = (id) => { setTodoList( todoList.map((todo) => todo.id === id ? { id: todo.id, contents: todo.contents, check: true } : todo)

setTodoList 로 다시 리스트를 만드는데

클릭 이벤트가 발생한 Ptag 의 id 는

{ id: todo.id, contents: todo.contents, check: true } 이렇게 세팅한다.

그리고 check: true 가 되면 css에 설정해둔 체크된 목록 디자인이 적용된다.

완료한 항목은 삭제하지 못한다.

​

​

2. [리스트 추가] 버튼

<Btn type="button" onClick={(e) => { const todo = { id: cnt, contents: word, check: false }; setCnt(cnt + 1); setTodoList([...todoList, todo]); setWord(""); }} >추가</Btn>

input에 내용을 입력하고 추가 버튼을 누르면 { id: cnt, contents: word, check: false }를 담고 있는 todo를 새로 만든다.

그리고 id 값이라 할 수 있는 cnt를 하나 증가 시키고,

기존의 todoList을 불러온 것에(...todoList) 새로 만든 todo를 추가해 setTodoList에 저장하고,

다시 input을 ""로 비운다.

​

3. 항목 삭제

우선 todoList 는 이렇게 보여진다.

{todoList && todoList.map((todo, index) => ( <> <p className={todo.check ? "true" : ""} isChecked key={todo.id} onClick={(e) => { todoCheck(todo.id); }} > {index + 1}: {todo.contents} {todo.check ? ( "" ) : ( <DeleteBtn type="button" onClick={(e) => { setTodoList(todoList.filter((item) => todo.id !== item.id)); }} > 🗑 </DeleteBtn>

완료 처리한 항목은 삭제가 안되게 하기 위해 todo.check가 되어있지 않은 항목을 확인해 거기에만 DeleteBtn을 보이게 했다.

​

삭제는 특정 원소를 배열에서 제거하기 위해 리액트 내장 함수인 filter를 사용했다.

이 함수는 배열에서 특정 조건이 만족하는 원소들만 추출하여 새로운 배열을 만들어준다.

​

​

[좋은 예]

TodoMain.js, TodoItem.js 두 개의 파일로 분리한다.

​

[TodoMain.js]

function TodoMain() { const [word, setWord] = useState(""); const [todoList, setTodoList] = useState([]); const [cnt, setCnt] = useState(1); return ( <> <Input type="text" value={word} onChange={(e) => { setWord(e.target.value); }} ></Input> <Btn type="button" onClick={(e) => { const todo = { id: cnt, contents: word }; setCnt(cnt + 1); setTodoList([...todoList, todo]); setWord(""); }} > 추가 </Btn> <br /> <br /> {todoList && todoList.map((todo, index) => <TodoItem todo={todo} index={index} />)} </> ); }

해당 컴포넌트에서 다른 컴포넌트(TodoItem)로 todo라는 값을 전달했다.

todo는 추가 버튼을 누를 시 생기는 변수인데

const todo = { id: cnt, contents: word }

리스트의 번째인지를 담은 cnt 값과 input에 입력한 리스트 내용인 word를 담고 있다.

​

input에 내용을 입력하고 버튼을 누르면 변수 todo를 만들고, cnt 값을 하나 증가시키고, todoList에 todo를 추가한다.

​

그리고 아래에서 todoList가 있으면 && todoList 안에 있는 값을 하나씩 꺼내어 TodoItem을 세팅하는데, TodoItem 컴포넌트에 todo 값을 todo라는 이름으로 넘겨준다.

TodoItem 컴포넌트를 보자.

​

[TodoItem.js]

function Test({ todo, index }) { const [check, setCheck] = useState(false); return ( <> <Ptag className={check ? "true" : ""} isChecked key={todo.id} onClick={(e) => { setCheck(true); }} > {index + 1}: {todo.contents} {check ? ( "" ) : ( <DeleteBtn type="button" onClick={(e) => {}}> 🗑 </DeleteBtn> )} </Ptag> </> ); }

​

Check 는 마찬가지로 리스트에서 해결 한 항목을 체크하는 기능이다.

className={check ? "true" : ""} 로 체크된 리스트면 className이 true가 되어

font-style: italic 와 같이 체크된 디자인을 적용 시켰다.(기능 위주의 정리라 디자인 코드는 생략했다.)

​

Ptag 태그를 누르면 setCheck(true)가 동작해 체크된 항목이 된다.

​

Ptag는 {index + 1}: {todo.contents} 를 나타낸다.

각각 리스트의 번호와 내용인데 내용은 넘겨받은 todo의 contents가 세팅되어있다. 만약 위에서 Ptag 태그를 눌러 체크한 항목이라면 {index + 1}: {todo.contents} 만 나타내고, 체크되지 않은 항목이라면 삭제 버튼도 보이게 구현했다.
