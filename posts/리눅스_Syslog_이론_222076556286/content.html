에러 났을 때 확인하면 왠만한 에러는 잡을 수 있다.

Syslog

앞서 살펴본 여러 로그들은 Syslog 라는 데몬 프로세스에 의해 관리되고 있는데!

/etc/rsyslog.conf

위 경로의 파일이 syslog가 관리하는 설정 파일이다.

​

rsyslog.conf 파일을 살펴보면 다음과 같은 형식으로 뭔가 주르륵 있을 것이다.

### RULES ### #kern.* /dev/consle

[Facility].[Level]

[Action]

Selector 필드

Action 필드(파일, 서버)

######

이 부분에 ip 주소 등이 있다면

원격지에 로그를 저장 할 수도 있는 것이다.

여러대의 컴퓨터 각각의 로그를 따로 저장하는 것이 아니라, 전부 모아서 중앙 서버에 저장시켜 통합으로 관리할 수도 있는 것!

Facility

Level

kern : 커널이 발생한 메시지

emerge : 일반적으로 모든 사용자에게 전달되는 패닉 상황 (블루스크린, 커널 패닉)

user : 사용자 프로세스

alert : 시스템 DB에 손상 등 즉시 수정해야되는 상황.

mail : mail 시스템 관련 서비스

crit : 하드 장치 오류 등 중대한 상황에 대한 경고

daemon : telnetd, ftpd, httpd과 관련된 서비스

err : 하등 장치 이외의 오류

auth : 로그인과 같은 인증 관련 서비스

warning : 경고 메시지. '어 이건 쫌..?'

그치만 일단 무시해도 됨

syslog : syslog 관련 서비스

notice : 특별한 처리가 필요할 수 있는 비오류 상황.. 쪼금 애매한?

cron : 예약작업 관련 서비스, crond, atd

info : 정보 메시지

* : 모든 서비스

debug : 프로그램 개발 또는 테스트 할 때 사용되는 메시지

none : 로그로 기록 X
